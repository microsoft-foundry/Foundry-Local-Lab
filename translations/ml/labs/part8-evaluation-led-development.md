# ഭാഗം 8: Foundry Local ഉപയോഗിച്ചുള്ള മൂല്യനിർണ്ണയ നേതൃത്വം ανάπτυξη

> **ലക്ഷ്യം:** നിങ്ങളുടെ AI ഏജന്റുകളെ ക്രമാതീതമായി പരിശോധന ചെയ്ത് സ്കോർ ചെയ്‌തുകൊണ്ട് ഒരു മൂല്യനിർണ്ണയ ഫ്രെയിമ്വർക്ക് നിർമ്മിക്കുക, പരീക്ഷിച്ചിരിക്കുന്ന ഏജന്റിന്റെയും വിധി നല്കുന്നയാളുടെയും ഇരുവർക്കും ഒരേ ലോക്കൽ മോഡൽ ഉപയോഗിച്ച്, ഇത് നിങ്ങൾക്ക് നിഷ്കർഷങ്ങളുമായി പ്രോംപ്റ്റുകളിൽ പരിഷ്കരണം നടത്താൻ സഹായിക്കും.

## എന്തിനു മൂല്യനിർണ്ണയ നേതൃത്വം ανάπτυξη?

AI ഏജന്റുകളെ നിർമ്മിക്കുമ്പോൾ "ഇത് ശരിയെ പോലെ തോന്നുന്നു" എന്ന് മാത്രം മതിയാകുന്നില്ല. **മൂല്യനിർണ്ണയ-നേതൃത്വ ανάπτυξη** ഏജന്റ് ഔട്ട്പുട്ടുകളെ കോഡായി പരിഗണിക്കുന്നു: ആദ്യം ടെസ്റ്റുകൾ എഴുതി, ഗുണമേന്മ അളക്കുകയും സ്കോർ ഒരു പരിധി വഴി വന്നപ്പോൾ മാത്രമേ ഷിപ്പ് ചെയ്യൂ.

Zava Creative Writer (ഭാഗം 7)ൽ, **എഡിറ്റർ ഏജന്റ്** ഇതിനകം ഒരു ലഘുലേഖ മൂല്യനിർണയകനായി പ്രവർത്തിക്കുന്നു; ഇത് ACCEPT അല്ലെങ്കിൽ REVISE തീരുമാനിക്കുന്നു. ഈ ലാബ് ആ രീതിയെ ഔദ്യോഗികമായി പ്രയോഗത്തിലാക്കുന്ന ആവർത്തന മൂല്യനിർണ്ണയ ഫ്രെയിമ്വർക്കായി മാറിച്ചെയ്യുന്നു, ഏജന്റിനെയും പൈപ്പ് ലൈനിനെയും ബാധിക്കുന്നു.

| പ്രശ്നം | പരിഹാരമ് |
|---------|----------|
| പ്രോംപ്റ്റ് മാറ്റങ്ങൾ നിശബ്ദമായി ഗുണമേന്മ തകരാറുണ്ടാക്കുന്നു | **ഗോൾഡൻ ഡാറ്റാസറ്റ്** പിന്‍വാതിൽ കണ്ടെത്തുന്നു |
| "ഒരു ഉദാഹരണത്തിൽ മാത്രം പ്രവർത്തിക്കുന്നു" എന്ന് മുൻഗണന | **വിവിധ ടെസ്റ്റ് കേസുകൾ** അതിരുകളെ വെളിപ്പെടുത്തുന്നു |
| വിഷയപരമായ ഗുണനിലവാര മൂല്യനിർണ്ണയം | **നിയമാധിഷ്ഠിത + LLM-ജഡ്ജ് സ്കോർ** സംഖ്യകൾ നൽകുന്നു |
| പ്രോംപ്റ്റ് വേരിയന്റുകൾ താരതമ്യം ചെയ്യാനുള്ള മാർഗം ഇല്ല | **പక్క - പക്കeval runs** സന്തുലിത സ്കോറുകൾ നൽകുന്നു |

## പ്രധാന ആശയങ്ങൾ

### 1. ഗോൾഡൻ ഡാറ്റാസെറ്റുകൾ

**ഗോൾഡൻ ഡാറ്റാസെറ്റ്** എന്നത് മുൻകൂട്ടി പ്രവചിക്കപ്പെട്ട ഔട്ട്പുട്ടുകൾ ഉള്ള കണ്ട്‌റൊൾ ചെയ്ത ടെസ്റ്റ് കേസുകളുടെ ചേരുവയാണ്. ഓരോ ടെസ്റ്റ് കേസും ഉൾക്കൊള്ളുന്നു:

- **ഇൻപുട്ട്**: ഏജന്റിന് അയക്കേണ്ട പ്രോംപ്റ്റും ചോദ്യവും
- **പ്രതീക്ഷിക്കപ്പെട്ട ഔട്ട്പുട്ട്**: ശരിയായോ ഉയർന്ന നിലവാരമുള്ള അതുപോലെ പ്രതീക്ഷിക്കുന്ന പ്രതികരണം (കീവേഡുകൾ, ഘടന, വസ്തുതകൾ)
- **വിഭാഗം**: റിപ്പോർട്ടിംഗിനുള്ള ഗ്രൂപ്പിംഗ് (ഉദാഹരണത്തിന് "വസ്തുനിഷ്ഠത", "ശബ്ദം", "പൂർണമാനം")

### 2. നിയമാധിഷ്ഠിത പരിശോധനകൾ

വേഗതയുള്ള, നിർണ്ണായകമായ പരിശോധനകൾ, LLM ആവശ്യമില്ലാത്തവ:

| പരിശോധന | ഇത് പരിശോധിക്കുന്നു |
|-----------|---------------------|
| **നീളം പരിധികൾ** | പ്രതികരണം വളരെ ചുരുണ്ടതോ (ലേസി) അല്ലെങ്കിൽ ഏറെ നീണ്ടതോ (വൃത്താന്തമാക്കൽ) അല്ല |
| **ആവശ്യമായ കീവേഡുകൾ** | പ്രതീക്ഷിച്ച പദങ്ങൾ അല്ലെങ്കിൽ ഘടകങ്ങൾ പരാമർശിക്കുന്നു |
| **ഫോർമാറ്റ് സാധുത** | JSON പാഴ്‌സக்கാവുന്നത്, മാര്ക്ക്ഡൗൺ ഹെഡറുകൾ ഉണ്ടായിരിക്കുന്നത് |
| **നിരോധിത ഉള്ളടക്കം** | no ഹല്ലൂസിനേറ്റഡ് ബ്രാന്റ് നെയിംസ്, മത്സരാർത്ഥി പരാമർശങ്ങൾ ഇല്ല |

### 3. LLM-ജഡ്ജ്

സ്വന്തം ഔട്ട്പുട്ടുകളെയും അല്ലെങ്കിൽ പല പ്രോംപ്റ്റ് വേരിയന്റുകളിൽ നിന്നുള്ള ഔട്ട്പുട്ടുകളെയും ഗ്രേഡ് ചെയ്യാൻ **അതേ ലോക്കൽ മോഡൽ** ഉപയോഗിക്കുക. ജഡ്ജ് സ്വീകരിക്കുന്നത്:

- മൂല ചോദ്യമ
- ഏജന്റിന്റെ പ്രതികരണം
- ഗ്രേഡിംഗ് മാനദണ്ഡങ്ങൾ

മറ്റിച്ച് സമ്പ്രതിർത്തിയ സ്കോർ തരുന്നത്. ഇത് ഭാഗം 7 ലെ എഡിറ്റർ മാതൃകയുടെ സമ്പൂര്‍ണ്ണ രൂപപ്പെടുത്തലാണ്.

### വ്യായാമം 2 - ഗോൾഡൻ ഡാറ്റാസെറ്റ് മനസ്സിലാക്കുക

മൂല്യനിർണ്ണയ സാമ്പിളിൽ നിർവ്വചിച്ച ടെസ്റ്റ് കേസുകൾ പരിശോധിക്കുക. ഓരോ ടെസ്റ്റ് കേസിനും:

```
{
  "input":    "What tools do I need to build a deck?",
  "expected": ["saw", "drill", "screws", "level"],
  "category": "product-recommendation"
}
```

**പരിശോധിക്കേണ്ട ചോദ്യങ്ങൾ:**
1. പ്രതീക്ഷിക്കപ്പെട്ട മൂല്യങ്ങൾ **കീവേഡുകൾ** ആയിരിക്കുന്നത് പൂർണ്ണ വാക്യങ്ങൾ ആകാതെ എന്തുകൊണ്ട്?
2. വിശ്വസനീയമായ മൂല്യനിർണ്ണയത്തിന് എത്ര ടെസ്റ്റ് കേസുകൾ ആവശ്യമാണ്?
3. നിങ്ങളുടെ അപ്ലിക്കേഷനിലേക്ക് നിങ്ങൾ എന്തേറെ വിഭാഗങ്ങൾ ചേർക്കും?

### വ്യായാമം 3 - നിയമാധിഷ്ഠിതറും LLM-ജഡ്ജും തമ്മിലുള്ള സ്കോറിംഗ് വ്യത്യാസം മനസ്സിലാക്കുക

മൂല്യനിർണ്ണയ ഫ്രെയിമ്വർക്ക് ഉപയോഗിക്കുന്നത് **രണ്ടു സ്കോറിംഗ് പാളികൾ**:

#### നിയമാധിഷ്ഠിത പരിശോധനകൾ (വേഗം, നിർണ്ണായക)

```
✓ Length: 50-500 words       → 1 point
✓ Keywords: 3/4 found        → 0.75 points  
✗ Forbidden: mentions "Home Depot" → 0 points
─────────────────────────────
Rule score: 0.58 / 1.0
```

#### LLM-ജഡ്ജ് (സൂക്ഷ്മത, ഗുണമേന്മ)

അതേ ലോക്കൽ മോഡൽ ജഡ്ജായി പ്രവർത്തിച്ച് ഘടനാപരമായ റൂബ്രിക് പാലിക്കുന്നു:

```
Rate this response on a scale of 1-5:
- 1: Completely wrong or irrelevant
- 2: Partially correct but missing key information
- 3: Adequate but could be improved
- 4: Good response with minor issues
- 5: Excellent, comprehensive, well-structured

Score: 4
Reasoning: The response correctly identifies all necessary tools
and provides practical advice, but could include safety equipment.
```

**പരിശോധിക്കേണ്ട ചോദ്യങ്ങൾ:**
1. എപ്പോഴാണ് നിങ്ങൾ നിയമാധിഷ്ഠിത പരിശോധനകൾ LLM-ജഡ്ജിനെക്കാൾ വിശ്വസിക്കും?
2. ഒരു മോഡൽ വിശ്വാസത്തോടെ സ്വന്തം ഔട്ട്പുട്ട് ജഡ്ജ് ചെയ്യാമോ? അതിന്റെ അവരോധങ്ങൾ എന്തൊക്കെയാണ്?
3. ഇത് ഭാഗം 7 ലെ എഡിറ്റർ ഏജന്റ് മാതൃകയുടെ താരതമ്യത്തിൽ എങ്ങിനെയാണ്?

### വ്യായാമം 4 - പ്രോംപ്റ്റ് വേരിയന്റുകൾ താരതമ്യം ചെയ്യുക

സാമ്പിള് **രണ്ട് പ്രോംപ്റ്റ് വേരിയന്റുകൾ** ഒരേ ടെസ്റ്റ് കേസുകളിൽ ഓടിക്കുന്നു:

| വേരിയന്റ് | സിസ്റ്റം പ്രോംപ്റ്റ് സ്റ്റൈൽ |
|-----------|-------------------------|
| **ബേസ്‌ലൈൻ** | പൊതുവായ: "നിങ്ങൾ ഒരു സഹായക സഹായി ആണ്" |
| **വിശേഷിപ്പിച്ച** | വിശദീകരിച്ച: "നിങ്ങൾ ഒരു Zava DIY വിദഗ്ധൻ ആണ്, പ്രത്യേക ഉൽപ്പന്നങ്ങൾ നിർദ്ദേശിക്കുകയും ഘട്ടം ഘട്ടം മാർഗ്ഗനിർദ്ദേശം നൽകുകയും ചെയ്യുന്നു" |

ഓടിക്കാൻ ശേഷം നിങ്ങൾക്ക് സ്കോർകാർഡ് ഇങ്ങനെ കാണാം:

```
╔══════════════════════════════════════════════════════════════╗
║                    EVALUATION SCORECARD                      ║
╠══════════════════════════════════════════════════════════════╣
║ Prompt Variant    │ Rule Score │ LLM Score │ Combined       ║
╠═══════════════════╪════════════╪═══════════╪════════════════╣
║ Baseline          │ 0.62       │ 3.2 / 5   │ 0.62           ║
║ Specialised       │ 0.81       │ 4.1 / 5   │ 0.81           ║
╚══════════════════════════════════════════════════════════════╝
```

**വ്യായാമങ്ങൾ:**
1. മൂല്യനിർണ്ണയം ഓടിച്ച് ഓരോ വേരിയന്റിന്റെയും സ്കോർ ശ്രദ്ധിക്കുക
2. വിശേഷിപ്പിച്ച പ്രോംപ്റ്റ് കൂടുതൽ നിഷ്കർഷമായാക്കുക. സ്കോർ മെച്ചപ്പെടുമോ?
3. മൂന്നാം പ്രോംപ്റ്റ് വേരിയന്റ് ചേർത്ത് മുഴുവനും താരതമ്യം ചെയ്യുക
4. മോഡൽ അലയാസ് മാറ്റി (e.g. `phi-4-mini` vs `phi-3.5-mini`) ഫലങ്ങൾ താരതമ്യം ചെയ്യുക

### വ്യായാമം 5 - നിങ്ങളുടെ സ്വന്തം ഏജന്റ് ഉൾപ്പെടുത്തുക

നിങ്ങളുടെ സ്വന്തം ഏജന്റുകൾക്കായി മൂല്യനിർണ്ണയ ഫ്രെയിമ്വർക്ക് മാതൃക ഉപയോഗിക്കുക:

1. **നിങ്ങളുടെ ഗോൾഡൻ ഡാറ്റാസെറ്റ് നിർവചിക്കുക**: 5 മുതൽ 10 വരെ ടെസ്റ്റ് കേസുകൾ കീവേഡുകളോടെ എഴുതുക
2. **നിങ്ങളുടെ സിസ്റ്റം പ്രോംപ്റ്റ് എഴുതി**: പരീക്ഷിക്കാൻ ആഗ്രഹിക്കുന്ന ഏജന്റ് നിർദ്ദേശങ്ങൾ
3. **മൂല്യനിർണ്ണയം ഓടിക്കുക**: അടിസ്ഥാന സ്കോറുകൾ നേടുക
4. **ഇട്ടറേറ്റ് ചെയ്യുക**: പ്രോംപ്റ്റ് തിരുത്തി വീണ്ടും ഓടിക്കുക, താരതമ്യം ചെയ്യുക
5. **പരിധി നിശ്ചയിക്കുക**: ഉദാഹരണം - "0.75 സംയുക്ത സ്കോർ താഴെയാണ് ഷിപ്പ് ചെയ്യരുത്"

### വ്യായാമം 6 - Zava എഡിറ്റർ മാതൃകയുമായി ബന്ധം

Zava Creative Writer-ന്റെ എഡിറ്റർ ഏജന്റിനെ നോക്കി ( `zava-creative-writer-local/src/api/agents/editor/editor.py` ):

```python
# എഡിറ്റർ പ്രൊഡക്ഷനിൽ ഒരു LLM-അസ്-ജഡ്ജ് ആണ്:
{"decision": "accept/revise", "editorFeedback": "...", "researchFeedback": "..."}
```

ഇത് ഭാഗം 8 ലെ LLM-ജഡ്ജുമായി **അതിനേത് തന്നെയാണ് ആശയം**, എന്നാൽ പ്രൊഡക്ഷൻ പൈപ്പ്ലൈനിൽ ഉൾപ്പെടുത്തി, ഓഫ്‌ലൈനിൽ ടെസ്റ്റ് സെക്രട്ടറി രൂപത്തിൽ അല്ല. ഇരുവട കൂടി:

- മോഡലിൽ നിന്നുള്ള ഘടനാപരമായ JSON ഔട്ട്പുട്ട് ഉപയോഗിക്കുന്നു.
- സിസ്റ്റം പ്രോംപ്റ്റിൽ നിർവചിച്ച ഗുണനിലവാര മാനദണ്ഡങ്ങൾ പ്രയോഗിക്കുന്നു.
- പാസ്സ്/ഫെയിൽ निर्णयം എടുക്കുന്നു.

**ഭേദം:** എഡിറ്റർ പ്രൊഡക്ഷനിൽ ഓടുന്നു (ഓരോ അഭ്യർത്ഥനയ്ക്കും). മൂല്യനിർണ്ണയ ഫ്രെയിംവർക്ക് വികസനത്തിൽ ഓടുന്നു (ഷിപ്പ് ചെയ്യുന്നതിന് മുൻപ്).
