# భాగం 8: Foundry Local తో మూల్యాంకన ఆధారిత అభివృద్ధి

> **లక్ష్యం:** పరీక్షలుగా మరియు స్కోర్లు కలిగిన మదింపు ఫ్రేమ్‌వర్క్‌ను నిర్మించడం, ఆజెంట్ యొక్క దిగుమతుల మోడల్‌ను పరీక్షించే మరియు న్యాయధ తీధించే మోడల్‌ను ఉపయోగించి, కంటెంట్‌ను జాగ్రత్తగా పంపిణీ చేయడానికి మౌలిక ఉత్ప్రేరకాలపై ఆరంభించి స్మార్ట్‌గా అభివృద్ధి చేయగలుగుతారు.

## మూల్యాంకన ఆధారిత అభివృద్ధి ఎందుకు?

AI ఏజెంట్లను నిర్మిస్తున్నప్పుడు, "సరే అనిపిస్తోంది" అనేది సరిపోదు. **మూల్యాంకన ఆధారిత అభివృద్ధి** ఏజెంట్ అవుట్పుట్లను కోడ్‌లాగా పరిగణిస్తుంది: మీరు ముందుగా పరీక్షలను రాయండి, నాణ్యతను కొలవండి, మరియు స్కోర్లు ఒక సరిహద్దు తీరినప్పుడు మాత్రమే పంపిణీ చేయండి.

Zava Creative Writer (భాగం 7)లో, **ఎడిటర్ ఏజెంట్** ఇప్పటికే ఒక తేలికపాటి మూల్యాంకకుడిగా పని చేస్తుంది; ఇది ACCEPT లేదా REVISE నిర్ణయిస్తుంది. ఈ ప్రయోగశాల ఆ నమూనాను పునరావృత మూల్యాంకన ఫ్రేమ్‌వర్క్‌గా రూపకల్పన చేస్తుంది, దీన్ని మీరు ఏ ఏజెంట్ లేదా పైప్‌లైన్‌కు వర్తించవచ్చు.

| సమస్య | పరిష్కారం |
|---------|----------|
| ప్రాంప్ట్ మార్పులు నిశ్శబ్దంగా నాణ్యత నాశనం చేస్తాయి | **గోల్డెన్ డేటాసెట్** తిరుగు పతనాలను పట్టుకుంటుంది |
| "ఒక ఉదాహరణలో పని చేస్తుంది" తతంగ భావం | **అనేక పరీక్ష కేసులు** ఎడ్జ్ కేసులను آشکارం చేస్తాయి |
| సబ్జెక్టివ్ నాణ్యత అంచనా | **నియమావళి ఆధారిత + LLM-న్యాయధీశుడు స్కోరు** సంఖ్యలు అందిస్తుంది |
| ప్రాంప్ట్ మార్పులు పోల్చే మార్గం లేదు | **పక్కపక్కన మూల్యాంకన పరుగులు** సమగ్ర స్కోరులతో |

## ముఖ్యమైన ఆలోచనలు

### 1. గోల్డెన్ డేటాసెట్స్

**గోల్డెన్ డేటాసెట్** అనేది ముందే సరిగ్గా తెలిసిన, అందుబాటులో ఉన్న అవుట్పుట్లతో కూడిన ఒక జాగ్రత్తగా ఎంపిక చేసిన పరీక్షా సెట్. ప్రతి పరీక్ష కేసు:

- **ఇన్పుట్**: ఏజెంట్‌కు పంపే ప్రాంప్ట్ లేదా ప్రశ్న
- **అనుకూల చెయ్యబడ్డ అవుట్పుట్**: సరైన లేదా అధిక నాణ్యత ప్రతిస్పందనలో ఉండవలసిన పదాలు, నిర్మాణం, వాస్తవాలు
- **వర్గం**: నివేదిక కోసం గుంపు (ఉదాహరణకు "ఫ్యాక్చువల్ ఖచ్చితత్వం", "శైలి", "పూర్తితనం")

### 2. నియమావళి ఆధారిత తనిఖీలు

వేగంగా, నిర్దిష్టమైన తనిఖీలు LLM అవసరం లేవు:

| తనిఖీ | ఇది ఏమి పరీక్షిస్తుంది |
|-------|--------------|
| **అమాంత్య పరిమితులు** | ప్రతిస్పందన చాలా తక్కువ (కోతపడటం) లేక చాలా ఎక్కువ (అధికవ్యాఖ్య) కాదు |
| **అవసరమైన కీలకపదాలు** | ప్రతిస్పందనలో అంచనా వేసిన పదాలు లేదా సంగ్రహాలు ఉన్నాయి |
| **ఫార్మాట్ చెలామణి** | JSON పార్స్ అవుతుంది, మార్క్‌డౌన్ హెడర్లు ఉన్నారు |
| **నిషేధిత విషయం** | భ్రమరాజించిన బ్రాండ్ పేర్లు లేక పోటీదారుల పేర్లు ఎక్కడా లేవు |

### 3. LLM-న్యాయధీశుడు

అనే **అదే స్థానిక మోడల్** ఉపయోగించి దాని స్వంత అవుట్పుట్ల (లేదా వేరే ప్రాంప్ట్ వేరియంట్ అవుట్పుట్లు) గ్రేడ్ చేయండి. న్యాయధీశుడు తీసుకునే అంశాలు:

- అసలు ప్రశ్న
- ఏజెంట్ ప్రతిస్పందన
- గ్రేడింగ్ ప్రమాణాలు

మరియు అవి ఒక నిర్మిత స్కోరు ఇవ్వగలవు. ఇది భాగం 7 లోని ఎడిటర్ నమూనా ని ప్రతిబింబిస్తాయ్ కానీ టెస్ట్ సూట్ పూర్వ వైఖరితో పరికించబడుతుంది.

### వ్యాయామం 2 - గోల్డెన్ డేటాసెట్‌ను అర్థం చేసుకోండి

మూల్యాంకన నమూనాలో నిర్వచించిన పరీక్ష కేసులను పరిశీలించండి. ప్రతి పరీక్ష కేసులో ఉంటుంది:

```
{
  "input":    "What tools do I need to build a deck?",
  "expected": ["saw", "drill", "screws", "level"],
  "category": "product-recommendation"
}
```

**సోచించాల్సిన ప్రశ్నలు:**
1. ఎందుకు అంచనా విలువలు పూర్తి వాక్యాల కంటే **కీలకపదాలు**?
2. విశ్వసనీయ మూల్యాంకన కోసం ఎంత పరీక్షా కేసులు అవసరం?
3. మీ స్వంత అనువర్తనానికి మీరు ఏ వర్గాలను జోడిస్తారు?

### వ్యాయామం 3 - నియమావళి ఆధారిత vs LLM-న్యాయధీశుడు స్కోరింగ్ అర్థం చేసుకోండి

మూల్యాంకన ఫ్రేమ్‌వర్క్ **రెండు స్కోరింగ్ పొరలు** ఉపయోగిస్తుంది:

#### నియమావళి ఆధారిత తనిఖీలు (వేగంగా, నిర్దిష్టంగా)

```
✓ Length: 50-500 words       → 1 point
✓ Keywords: 3/4 found        → 0.75 points  
✗ Forbidden: mentions "Home Depot" → 0 points
─────────────────────────────
Rule score: 0.58 / 1.0
```

#### LLM-న్యాయధీశుడు (సూక్ష్మ, నాణ్యతాపూర్ణ)

అదే స్థానిక మోడల్ నిర్మితమైన రూబ్రిక్‌తో న్యాయధీశుడుగా పనిచేస్తుంది:

```
Rate this response on a scale of 1-5:
- 1: Completely wrong or irrelevant
- 2: Partially correct but missing key information
- 3: Adequate but could be improved
- 4: Good response with minor issues
- 5: Excellent, comprehensive, well-structured

Score: 4
Reasoning: The response correctly identifies all necessary tools
and provides practical advice, but could include safety equipment.
```

**సోచించాల్సిన ప్రశ్నలు:**
1. మీరు ఎప్పుడు నియమావళి తనిఖీలను LLM-న్యాయధీశుడి కంటే విశ్వసిస్తారు?
2. ఒక మోడల్ తన స్వంత అవుట్పుట్ను విశ్వసనీయంగా న్యాయధీశించగలదా? పరిమితులు ఏమిటి?
3. ఈతొ భాగం 7 లోని ఎడిటర్ ఏజెంట్ నమూనాతో పోల్చితే ఎలా ఉంది?

### వ్యాయామం 4 - ప్రాంప్ట్ వేరియంట్లను పోల్చండి

నమూనా అదే పరీక్షా కేసులపై **రెండు ప్రాంప్ట్ వేరియంట్లు** నడుపుతుంది:

| వేరియంట్ | సిస్టమ్ ప్రాంప్ట్ శైలి |
|---------|-------------------|
| **బేస్‌లైన్** | సాధారణ: "మీరు ఒక ఉపయోగకరమైన సహాయకుడు" |
| **ప్రత్యేకీకరించి** | వివరించినది: "మీరు ఒక Zava DIY నిపుణుడు, నిర్దిష్ట ఉత్పత్తులను సూచిస్తూ దశల వారీ సూచనలు అందించేవారు" |

నడిపిన తర్వాత, మీరు ఇలా ఒక స్కోర్కార్డు చూడగలరు:

```
╔══════════════════════════════════════════════════════════════╗
║                    EVALUATION SCORECARD                      ║
╠══════════════════════════════════════════════════════════════╣
║ Prompt Variant    │ Rule Score │ LLM Score │ Combined       ║
╠═══════════════════╪════════════╪═══════════╪════════════════╣
║ Baseline          │ 0.62       │ 3.2 / 5   │ 0.62           ║
║ Specialised       │ 0.81       │ 4.1 / 5   │ 0.81           ║
╚══════════════════════════════════════════════════════════════╝
```

**వ్యాయామాలు:**
1. మూల్యాంకన నడిపి ప్రతి వేరియంట్ కోసం స్కోర్లు గమనించండి
2. ప్రత్యేకీకరించిన ప్రాంప్ట్‌ను మరింత సహజమైనదిగా మార్చండి. స్కోరు మెరుగవుతున్నదా?
3. మూడవ ప్రాంప్ట్ వేరియంట్ జోడించి మూడు అన్ని వేరియంట్లను పోల్చండి.
4. మోడల్ అలియాస్ (ఉదా: `phi-4-mini` vs `phi-3.5-mini`) మార్చి ఫలితాలను పోల్చండి.

### వ్యాయామం 5 - మీ స్వంత ఏజెంట్‌కు మూల్యాంకన అన్వయించండి

మూల్యాంకన ఫ్రేమ్‌వర్క్‌ను మీ ఏజెంట్ల కోసం టెంప్లేట్‌గా ఉపయోగించండి:

1. **మీ గోల్డెన్ డేటాసెట్ నిర్వచించండి**: 5 నుండి 10 పరీక్ష కేసులను ఆశించిన కీలకపదాలతో రాయండి.
2. **మీ సిస్టమ్ ప్రాంప్ట్ రాయండి**: మీరు పరీక్షించదలచుకున్న ఏజెంట్ సూచనలు.
3. **మూల్యాంకనం నడపండి**: బేస్‌లైన్ స్కోర్లు పొందండి.
4. **పునరావృతం చేయండి**: ప్రాంప్ట్‌ను సవరించండి, తిరిగి నడిపించి, పోల్చండి.
5. **సరిహద్దు సెట్ చేయండి**: ఉదా: "0.75 కంటే తక్కువ కలిపిన స్కోరు వద్ద పంపవద్దు".

### వ్యాయామం 6 - Zava ఎడిటర్ నమూనాతో సంబంధం

Zava Creative Writer యొక్క ఎడిటర్ ఏజెంట్ని తిరిగి చూడండి (`zava-creative-writer-local/src/api/agents/editor/editor.py`):

```python
# ఎడిటర్ ప్రొడక్షన్లో LLM-అస్-జడ్జ్ గా ఉంది:
{"decision": "accept/revise", "editorFeedback": "...", "researchFeedback": "..."}
```

ఇది భాగం 8 లోని LLM-న్యాయధీశుడి **అదే భావన**, కానీ తయారీ పైప్‌లైన్‌లో ఉంచబడింది, ఆఫ్‌లైన్ టెస్ట్ సూట్‌ని కాకుండా. రెండూ:

- మోడల్ నుండి నిర్మిత JSON అవుట్పుట్‌ను ఉపయోగిస్తాయి.
- సిస్టమ్ ప్రాంప్ట్‌లో నిర్వచించిన నాణ్యత ప్రమాణాలు వర్తించాయి.
- సరైన/పోరుపు నిర్ణయం తీసుకుంటాయి.

**తేడా:** ఎడిటర్ తయారీ లో నడుస్తుంది (ప్రతి అభ్యర్థనపై). మూల్యాంకన ఫ్రేమ్‌వర్క్ అభివృద్ధిలో నడుస్తుంది (మీరు పంపిణీ చేసేముందు).

## తదుపరి దశలు

- **పరిమాణాన్ని పెంచండి**: మీ గోల్డెన్ డేటాసెట్‌లో మరిన్ని పరీక్ష కేసులు మరియు వర్గాలను జోడించండి.
- **స్వయంచాలకీకరించండి**: మీ CI/CD పైప్‌లైన్‌లో మూల్యాంకనాన్ని సమగ్రపరచండి.
- **అత్యాధునిక న్యాయధీశులు**: చిన్న మోడల్ అవుట్పుట్‌ను పరీక్షిస్తూ పెద్ద మోడల్‌ను న్యాయధీశుగా ఉపయోగించండి.
- **కాలక్రమంగా ట్రాక్ చేయండి**: మూల్యాంకన ఫలితాలను పొంది ప్రాంప్ట్ మరియు మోడల్ వెర్షన్లను పోల్చండి.
