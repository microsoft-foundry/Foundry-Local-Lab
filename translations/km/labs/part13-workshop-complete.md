# ផ្នែក ១៣៖ កិច្ចសិក្សាអប់រំនេះបានបញ្ចប់

> **អបអរសាទរ!** អ្នកបានបញ្ចប់វគ្គសិក្សា Foundry Local ពេញលេញ។

អ្នកបានផ្លាស់ប្តូរពីការដំឡើង CLI ទៅការសាងសង់កម្មវិធីជជែក ស្ថាប័ន RAG ប្រព័ន្ធភ្នាក់ងារច្រើន ការបំលែងសំឡេងទៅអត្ថបទ ការបញ្ចូលម៉ូដែលផ្ទាល់ខ្លួន ការមើលឃើញការហៅឧបករណ៍ និងការតភ្ជាប់ UI វេបសាយសម្រាប់កម្មវិធីភ្នាក់ងារច្រើនរបស់អ្នក។ អ្វីៗទាំងនោះដំណើរការពេញលេញលើឧបករណ៍របស់អ្នក។

---

## អ្វីដែលអ្នកបានសាងសង់

| ផ្នែក | អ្វីដែលអ្នកបានសាងសង់ |
|------|---------------|
| 1 | បានដំឡើង Foundry Local ស្វែងយល់ម៉ូដែលតាម CLI |
| 2 | ចំណេះដឹង SDK API របស់ Foundry Local សម្រាប់សេវាកម្ម កាតាឡុក កាឆេន និងការគ្រប់គ្រងម៉ូដែល |
| 3 | តភ្ជាប់ពី Python, JavaScript និង C# ប្រើ SDK ជាមួយ OpenAI |
| 4 | សាងសង់ស្ថាប័ន RAG ជាមួយការទាញយកចំណេះដឹងក្នុងតំបន់ |
| 5 | បង្កើតភ្នាក់ងារ AI មានបុគ្គលិកលក្ខណៈ និងលទ្ធផលដូចសំណុំ |
| 6 | ប្រតិបត្តិស្ថាប័នភ្នាក់ងារច្រើនជាមួយរង្វង់បញ្ចូលមតិយោបល់ |
| 7 | ស្វែងយល់កម្មវិធីសម្រួលផលិតកម្មមួយ Zava Creative Writer |
| 8 | សាងសង់ដំណើរការអភិវឌ្ឍន៍ដោយការវាយតម្លៃសម្រាប់ភ្នាក់ងារ |
| 9 | បំលែងសំឡេងជាអត្ថបទជាមួយ Whisper ដំណើរការនៅលើឧបករណ៍ |
| 10 | បញ្ចេញនិងដំណើរការម៉ូដែល Hugging Face ផ្ទាល់ខ្លួនជាមួយ ONNX Runtime GenAI |
| 11 | អនុញ្ញាតម៉ូដែលក្នុងតំបន់ហៅមុខងារខាងក្រៅដោយការហៅឧបករណ៍ |
| 12 | សាងសង់ UI វេបសាយសម្រាប់ Zava Creative Writer ជាមួយការបញ្ច្រាសបន្តពេញលេញ |

## គំនិតបន្ថែម

- បញ្ចូលការហៅឧបករណ៍ជាមួយភ្នាក់ងារសម្រាប់បង្កើតដំណើរការអូតូម៉ាទិក
- សួរទាញយកទិន្នន័យក្នុងមូលដ្ឋានទិន្នន័យក្នុងតំបន់ ឬហៅ REST APIs ខាងក្នុងពីឧបករណ៍របស់អ្នក
- សាកល្បងម៉ូដែលផ្សេងៗ (`phi-4-mini`, `deepseek-r1-7b`) សម្រាប់ប្រៀបធៀបទំហំ និងល្បឿន
- បន្ថែមការផ្ទៀងផ្ទាត់ ឬកំណត់អត្រាប្រើប្រាស់ទៅម៉ាស៊ីនបម្រើវេបសាយ Zava Writer របស់អ្នក
- បង្កើតកម្មវិធីភ្នាក់ងារច្រើនរបស់អ្នកសម្រាប់ដែនកំណត់ដែលអ្នកចាប់អារម្មណ៍
- បញ្ចេញទៅពពកដោយប្ដូរ Foundry Local ទៅ Azure AI Foundry កូដដូចគ្នា ប៉ុន្តែ endpoint ផ្សេង
