# Μέρος 8: Ανάπτυξη Με Οδηγό την Αξιολόγηση με το Foundry Local

> **Στόχος:** Δημιουργήστε ένα πλαίσιο αξιολόγησης που δοκιμάζει συστηματικά και βαθμολογεί τους AI παράγοντές σας, χρησιμοποιώντας το ίδιο τοπικό μοντέλο τόσο ως παράγοντα που δοκιμάζεται όσο και ως κριτή, ώστε να μπορείτε να επαναλάβετε προτροπές με αυτοπεποίθηση πριν την αποστολή.

## Γιατί Ανάπτυξη με Οδηγό την Αξιολόγηση;

Κατά την κατασκευή AI παραγόντων, το "φαίνεται σωστό" δεν αρκεί. Η **ανάπτυξη με οδηγό την αξιολόγηση** αντιμετωπίζει τα αποτελέσματα του παράγοντα όπως τον κώδικα: γράφετε πρώτα δοκιμές, μετράτε την ποιότητα, και αποστέλλετε μόνο όταν οι βαθμολογίες πληρούν ένα όριο.

Στον Zava Creative Writer (Μέρος 7), ο **Editor agent** ήδη λειτουργεί ως ελαφρύς αξιολογητής· αποφασίζει ΕΓΚΡΙΣΗ ή ΕΠΙΣΚΟΠΗΣΗ. Αυτό το εργαστήριο τυποποιεί αυτό το μοτίβο σε ένα επαναλαμβανόμενο πλαίσιο αξιολόγησης που μπορείτε να εφαρμόσετε σε οποιονδήποτε παράγοντα ή ροή εργασίας.

| Πρόβλημα | Λύση |
|---------|----------|
| Οι αλλαγές στην προτροπή σπάζουν αθόρυβα την ποιότητα | Το **χρυσό σύνολο δεδομένων** εντοπίζει οπισθοδρομήσεις |
| Προκατάληψη "λειτουργεί σε ένα παράδειγμα" | **Πολλαπλά τεστ** αποκαλύπτουν περιπτώσεις οριακών καταστάσεων |
| Υποκειμενική αξιολόγηση ποιότητας | **Κανόνες + αξιολόγηση με LLM-κριτή** παρέχουν αριθμούς |
| Δεν υπάρχει τρόπος σύγκρισης παραλλαγών προτροπών | **Παράλληλες εκτελέσεις αξιολόγησης** με συνολικές βαθμολογίες |

## Κύριες Έννοιες

### 1. Χρυσά Σύνολα Δεδομένων

Ένα **χρυσό σύνολο δεδομένων** είναι ένα επιλεγμένο σύνολο δοκιμαστικών περιπτώσεων με γνωστά αναμενόμενα αποτελέσματα. Κάθε περίπτωση δοκιμής περιλαμβάνει:

- **Είσοδο:** Την προτροπή ή ερώτηση προς τον παράγοντα
- **Αναμενόμενο αποτέλεσμα:** Τι πρέπει να περιέχει μια σωστή ή υψηλής ποιότητας απάντηση (λέξεις-κλειδιά, δομή, γεγονότα)
- **Κατηγορία:** Ομαδοποίηση για αναφορά (π.χ. "εγκυρότητα δεδομένων", "τόνος", "πληρότητα")

### 2. Έλεγχοι με Κανόνες

Γρήγοροι, ντετερμινιστικοί έλεγχοι που δεν απαιτούν LLM:

| Έλεγχος | Τι Ελέγχει |
|-------|--------------|
| **Περιορισμοί μήκους** | Η απάντηση δεν είναι πολύ σύντομη (τεμπέλικη) ή πολύ μακροσκελής (λογομαχία) |
| **Απαιτούμενες λέξεις-κλειδιά** | Η απάντηση αναφέρει αναμενόμερους όρους ή οντότητες |
| **Επικύρωση μορφής** | Το JSON είναι αναγνώσιμο, υπάρχουν επικεφαλίδες Markdown |
| **Απαγορευμένο περιεχόμενο** | Δεν υπάρχουν φανταστικές επωνυμίες, ούτε αναφορές ανταγωνιστών |

### 3. LLM-κριτής

Χρησιμοποιήστε το **ίδιο τοπικό μοντέλο** για να βαθμολογήσει τα δικά του αποτελέσματα (ή αποτελέσματα από διαφορετική παραλλαγή προτροπής). Ο κριτής λαμβάνει:

- Την αρχική ερώτηση
- Την απάντηση του παράγοντα
- Κριτήρια βαθμολόγησης

Και επιστρέφει μια δομημένη βαθμολογία. Αυτό αντικατοπτρίζει το μοτίβο Editor από το Μέρος 7 αλλά εφαρμόζεται συστηματικά σε ένα σύνολο δοκιμών.

### Άσκηση 2 - Κατανόηση του Χρυσού Συνόλου

Εξετάστε τις δοκιμαστικές περιπτώσεις στον προτύπο αξιολόγησης. Κάθε περίπτωση έχει:

```
{
  "input":    "What tools do I need to build a deck?",
  "expected": ["saw", "drill", "screws", "level"],
  "category": "product-recommendation"
}
```

**Ερωτήσεις προς σκέψη:**
1. Γιατί οι αναμενόμενες τιμές είναι **λέξεις-κλειδιά** αντί για πλήρεις προτάσεις;
2. Πόσες δοκιμαστικές περιπτώσεις χρειάζεστε για αξιόπιστη αξιολόγηση;
3. Ποιες κατηγορίες θα προσθέτατε για τη δική σας εφαρμογή;

### Άσκηση 3 - Κατανόηση της Βαθμολόγησης με Κανόνες vs LLM-κριτή

Το πλαίσιο αξιολόγησης χρησιμοποιεί **δύο επίπεδα βαθμολόγησης**:

#### Έλεγχοι με Κανόνες (γρήγοροι, ντετερμινιστικοί)

```
✓ Length: 50-500 words       → 1 point
✓ Keywords: 3/4 found        → 0.75 points  
✗ Forbidden: mentions "Home Depot" → 0 points
─────────────────────────────
Rule score: 0.58 / 1.0
```

#### LLM-κριτής (λεπτομερής, ποιοτική)

Το ίδιο τοπικό μοντέλο λειτουργεί ως κριτής με δομημένο πλαίσιο:

```
Rate this response on a scale of 1-5:
- 1: Completely wrong or irrelevant
- 2: Partially correct but missing key information
- 3: Adequate but could be improved
- 4: Good response with minor issues
- 5: Excellent, comprehensive, well-structured

Score: 4
Reasoning: The response correctly identifies all necessary tools
and provides practical advice, but could include safety equipment.
```

**Ερωτήσεις προς σκέψη:**
1. Πότε θα εμπιστευόσασταν ελέγχους με κανόνες αντί για LLM-κριτή;
2. Μπορεί ένα μοντέλο να κρίνει αξιόπιστα τη δική του έξοδο; Ποιες είναι οι περιορισμοί;
3. Πώς συγκρίνεται αυτό με το μοτίβο Editor agent από το Μέρος 7;

### Άσκηση 4 - Σύγκριση Παραλλαγών Προτροπών

Το παράδειγμα εκτελεί **δύο παραλλαγές προτροπών** στις ίδιες δοκιμές:

| Παραλλαγή | Στυλ συστήματος προτροπής |
|---------|-------------------|
| **Βασική** | Γενική: "Είσαι ένας βοηθητικός βοηθός" |
| **Εξειδικευμένη** | Λεπτομερής: "Είσαι ειδικός στο Zava DIY που προτείνει συγκεκριμένα προϊόντα και παρέχει βήμα-βήμα καθοδήγηση" |

Μετά την εκτέλεση, θα δείτε ένα scorecard όπως:

```
╔══════════════════════════════════════════════════════════════╗
║                    EVALUATION SCORECARD                      ║
╠══════════════════════════════════════════════════════════════╣
║ Prompt Variant    │ Rule Score │ LLM Score │ Combined       ║
╠═══════════════════╪════════════╪═══════════╪════════════════╣
║ Baseline          │ 0.62       │ 3.2 / 5   │ 0.62           ║
║ Specialised       │ 0.81       │ 4.1 / 5   │ 0.81           ║
╚══════════════════════════════════════════════════════════════╝
```

**Ασκήσεις:**
1. Τρέξτε την αξιολόγηση και σημειώστε τις βαθμολογίες για κάθε παραλλαγή
2. Τροποποιήστε την εξειδικευμένη προτροπή να γίνει ακόμα πιο συγκεκριμένη. Βελτιώνεται η βαθμολογία;
3. Προσθέστε τρίτη παραλλαγή προτροπής και συγκρίνετε και τις τρεις.
4. Δοκιμάστε να αλλάξετε το ψευδώνυμο μοντέλου (π.χ. `phi-4-mini` έναντι `phi-3.5-mini`) και συγκρίνετε αποτελέσματα.

### Άσκηση 5 - Εφαρμογή Αξιολόγησης στον Δικό Σας Παράγοντα

Χρησιμοποιήστε το πλαίσιο αξιολόγησης ως πρότυπο για τους δικούς σας παράγοντες:

1. **Ορίστε το χρυσό σύνολο σας:** γράψτε 5 έως 10 δοκιμές με αναμενόμενες λέξεις-κλειδιά.
2. **Γράψτε το σύστημα προτροπής σας:** τις οδηγίες παράγοντα που θέλετε να δοκιμάσετε.
3. **Τρέξτε την αξιολόγηση:** πάρετε τις βασικές βαθμολογίες.
4. **Επαναλάβετε:** τροποποιήστε την προτροπή, επανεκτελέστε, συγκρίνετε.
5. **Θέστε ένα όριο:** π.χ. "μην αποστέλλετε κάτω από 0.75 συνολική βαθμολογία".

### Άσκηση 6 - Σύνδεση με το Μοτίβο Editor του Zava

Ρίξτε μια ματιά στο Editor agent του Zava Creative Writer (`zava-creative-writer-local/src/api/agents/editor/editor.py`):

```python
# Ο επεξεργαστής είναι ένα LLM-ως-δικαστής σε παραγωγή:
{"decision": "accept/revise", "editorFeedback": "...", "researchFeedback": "..."}
```

Αυτή είναι η **ίδια έννοια** με το LLM-κριτή του Μέρους 8, αλλά ενσωματωμένη στη γραμμή παραγωγής αντί για offline σύνολο δοκιμών. Και τα δύο μοτίβα:

- Χρησιμοποιούν δομημένη JSON έξοδο από το μοντέλο.
- Εφαρμόζουν κριτήρια ποιότητας ορισμένα στην προτροπή συστήματος.
- Παίρνουν απόφαση πέρα-ή-αποτυχίας.

**Η διαφορά:** Ο editor τρέχει στην παραγωγή (σε κάθε αίτημα). Το πλαίσιο αξιολόγησης τρέχει στην ανάπτυξη (πριν την αποστολή).

|---------|----------|
| **Χρυσά σύνολα** | Επιλέξτε δοκιμές νωρίς· είναι οι δοκιμές οπισθοδρόμησης για AI |
