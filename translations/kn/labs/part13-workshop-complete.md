# ಭಾಗ 13: ಕಾರ್ಯಾಗಾರ ಪೂರ್ಣಗೊಂಡಿತು

> **ಅಭಿನಂದನೆಗಳು!** ನೀವು ಸಂಪೂರ್ಣ Foundry Local ಕಾರ್ಯಾಗಾರವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ್ದೀರಿ.

ನೀವು CLI ಅನ್ನು ಸ್ಥಾಪಿಸುವುದರಿಂದ ಪ್ರಾರಂಭಿಸಿ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ಗಳು, RAG ಪೈಪ್ಲೈನ್‌ಗಳು, ಬಹು-ಏಜೆಂಟ್ ಸಿಸ್ಟಮ್‌ಗಳು, ಶಬ್ದದಿಂದ ಪಠ್ಯಕ್ಕೆ ವರ್ಗಾವಣೆ, ನಿಮ್ಮದೇ ಸ್ವಂತ ಕಸ್ಟಮ್ ಮಾದರಿಗಳನ್ನು ಸಂಯೋಜಿಸುವುದು, ಉಪಕರಣ ಕರೆ ಮಾಡಲು ಅವಕಾಶ ನೀಡುವುದು, ಮತ್ತು ನಿಮ್ಮ ಬಹು-ಏಜೆಂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಲೈವ್ ವೆಬ್ UI ಅನ್ನು ಜೋಡಿಸುವುದರವರೆಗೆ ಎಲ್ಲದನ್ನೂ ನಿಮ್ಮ ಸಾಧನದಲ್ಲಿ ಸಂಪೂರ್ಣವಾಗಿ ನಿರ್ವಹಿಸಿದ್ದೀರಿ.

---

## ನೀವು ನಿರ್ಮಿಸಿದವು

| ಭಾಗ | ನೀವು ನಿರ್ಮಿಸಿದವು |
|------|---------------|
| 1 | Foundry Local ಅನ್ನು ಸ್ಥಾಪಿಸಿ, CLI ಮೂಲಕ ಮಾದರಿಗಳನ್ನು ಅದ್ದಾಡಿ |
| 2 | ಸರ್ವೀಸ್, ಕ್ಯಾಟಲಾಗ್, ಕ್ಯಾಶೆ ಮತ್ತು ಮಾದರಿ ನಿರ್ವಹಣೆಗೆ Foundry Local SDK API ಗೆ ಪರಿಣತಿ ಪಡೆದಿರಿ |
| 3 | SDK ಬಳಸಿ Python, JavaScript ಮತ್ತು C# ಮೂಲಕ OpenAI ಜೊತೆಗೆ ಸಂಪರ್ಕ ಸ್ಥಾಪನೆ ಮಾಡಿದರು |
| 4 | ಸ್ಥಳೀಯ ಜ್ಞಾನ ಪಡೆಯುವಿಕೆ ಮೂಲಕ RAG ಪೈಪ್ಲೈನ್ ನಿರ್ಮಿಸಿದೆವು |
| 5 | ಪರ್ಸೋನಾಸ್ ಮತ್ತು ರಚನಾತ್ಮಕ ಔಟ್‌ಪುಟ್‌ನೊಂದಿಗೆ AI ಏಜೆಂಟ್‌ಗಳನ್ನು ರಚಿಸಿದರು |
| 6 | ಪ್ರತಿಕ್ರಿಯಾ ಲೂಪ್ಗಳೊಂದಿಗೆ ಬಹು-ಏಜೆಂಟ್ ಪೈಪ್ಲೈನ್‌ಗಳನ್ನು ಸಂಯೋಜಿಸಿದರು |
| 7 | ಉತ್ಪಾದನಾ ಕ್ಯಾಪ್‌ಸ್ಟೋನ್ ಅಪ್ಲಿಕೇಶನ್, Zava ಕ್ರಿಯೇಟಿವ್ ರೈಟರ್ ಅನ್ನು ವೀಕ್ಷಿಸಿದರು |
| 8 | ಏಜೆಂಟ್‌ಗಳಿಗೆ ಮೌಲ್ಯಮಾಪನಾಧಾರಿತ ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯವಾಹಿಗಳನ್ನು ರಚಿಸಿದರು |
| 9 | ಸ್ಥಳೀಯ ಸಾಧನದಲ್ಲಿ Whisper ಬಳಸಿ ಶಬ್ದದಿಂದ ಪಠ್ಯಕ್ಕೆ ವರ್ಗಾವಣೆ ಮಾಡಿದರು |
| 10 | ONNX Runtime GenAI ಸಹಿತ ಕಸ್ಟಮ್ Hugging Face ಮಾದರಿಯನ್ನು ಸಂಯೋಜಿಸಿ ಚಾಲನೆ ಮಾಡಿದರು |
| 11 | ಉಪಕರಣ ಕರೆ ಮೂಲಕ ಸ್ಥಳೀಯ ಮಾದರಿಗಳಿಗೆ ಔಟ್‌ಸೈಡ್ ಫಂಕ್ಷನ್‌ಗಳ ಕರೆ ಮಾಡಲು ಅನುಮತಿಸಿದರು |
| 12 | ವಾಸ್ತವಿಕ ಕಾಲದ ಸ್ಟ್ರೀಮಿಂಗ್ ಸಹಿತ Zava ಕ್ರಿಯೇಟಿವ್ ರೈಟರ್‌ಗೆ ಬ್ರೌಸರ್ ಆಧಾರಿತ UI ನಿರ್ಮಿಸಿದರು |

---

## ಮುಂದಿನ ಆಲೋಚನೆಗಳು

- ಸ್ವಯಂಚಾಲಿತ ಕಾರ್ಯವಾಹಿಗಳನ್ನು ನಿರ್ಮಿಸಲು ಏಜೆಂಟ್‌ಗಳ ಜೊತೆಗೆ ಉಪಕರಣ ಕರೆ ಸಂಯೋಜಿಸಿ
- ನಿಮ್ಮ ಉಪಕರಣಗಳಿಂದ ಸ್ಥಳೀಯ ಡೇಟಾಬೇಸ್ ಅಥವಾ ಆಂತರಿಕ REST API ಗಳನ್ನು ಪ್ರಶ್ನಿಸಿ ಕರೆಮಾಡಿ
- ಗುಣಮಟ್ಟ ಮತ್ತು ವೇಗವನ್ನು ಹೋಲಿಸಲು ವಿಭಿನ್ನ ಮಾದರಿಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ (`phi-4-mini`, `deepseek-r1-7b`)
- ನಿಮ್ಮ Zava ರೈಟರ್ ವೆಬ್ ಸರ್ವರ್‌ಗೆ ಗೋಪ್ಯತೆ ಅಥವಾ ಪ್ರಮಾಣೀಕರಣವನ್ನು ಸೇರಿಸಿ
- ನೀವು ಆಸಕ್ತಿಯಾದ ಕ್ಷೇತ್ರಕ್ಕೆ ಸ್ವಂತ ಬಹು-ಏಜೆಂಟ್ ಅಪ್ಲಿಕೇಶನ್ ರಚಿಸಿ
- Foundry Local ಅನ್ನು Azure AI Foundry ಗೆ ಬದಲಿ ಹಾಕಿ ಕ್ಲೌಡ್‌ಗೆ ನಿಯೋಜಿಸಿ, ಅದೇ ಕೋಡ್, ವಿಭಿನ್ನ ಎಂಡ್‌ಪಾಯಿಂಟ್
