# Part 13: 워크숍 완료

> **축하합니다!** Foundry Local 워크숍을 모두 완료하셨습니다.

CLI 설치부터 챗 애플리케이션 구축, RAG 파이프라인, 다중 에이전트 시스템, 음성-텍스트 전사, 맞춤형 모델 컴파일, 도구 호출 활성화, 다중 에이전트 애플리케이션을 위한 실시간 웹 UI 연결까지 모두 진행하셨습니다. 모든 것이 완전히 귀하의 장치에서 실행됩니다.

---

## 당신이 만든 것

| Part | 당신이 만든 것 |
|------|---------------|
| 1 | Foundry Local 설치, CLI를 통한 모델 탐색 |
| 2 | 서비스, 카탈로그, 캐시 및 모델 관리를 위한 Foundry Local SDK API 마스터 |
| 3 | Python, JavaScript, C#에서 SDK와 OpenAI를 사용해 연결 |
| 4 | 로컬 지식 검색을 활용한 RAG 파이프라인 구축 |
| 5 | 페르소나와 구조화된 출력 기능이 있는 AI 에이전트 생성 |
| 6 | 피드백 루프가 포함된 다중 에이전트 파이프라인 조율 |
| 7 | 프로덕션 캡스톤 애플리케이션인 Zava Creative Writer 탐색 |
| 8 | 에이전트 평가 주도 개발 워크플로 구축 |
| 9 | Whisper로 오디오 전사, 장치 내 음성-텍스트 변환 |
| 10 | ONNX Runtime GenAI로 맞춤형 Hugging Face 모델 컴파일 및 실행 |
| 11 | 도구 호출을 사용하여 로컬 모델이 외부 함수 호출 가능하도록 함 |
| 12 | 실시간 스트리밍 기능을 가진 Zava Creative Writer 웹 기반 UI 구축 |

---

## 추가 아이디어

- 도구 호출과 에이전트를 결합해 자율 워크플로 구축
- 도구에서 로컬 데이터베이스 쿼리 또는 내부 REST API 호출
- 다양한 모델 (`phi-4-mini`, `deepseek-r1-7b`) 시험 사용해 품질과 속도 비교
- Zava Writer 웹 서버에 인증 또는 속도 제한 추가
- 관심 있는 도메인에 맞는 자체 다중 에이전트 애플리케이션 생성
- Foundry Local을 Azure AI Foundry로 교체해 클라우드에 배포, 코드 동일, 엔드포인트만 다름
