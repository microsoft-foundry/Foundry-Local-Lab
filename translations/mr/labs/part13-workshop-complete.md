# भाग 13: कार्यशाळा पूर्ण

> **अभिनंदन!** आपण फाउंड्री लोकल कार्यशाळा पूर्ण केली आहे.

आपण CLI ची स्थापना करण्या पासून चॅट अ‍ॅप्लिकेशन तयार करणे, RAG पाइपलाइन, मल्टी-एजंट सिस्टम, स्पीच-टू-टेक्स्ट ट्रान्सक्रिप्शन, स्वतःचे सानुकूल मॉडेल कंपाईल करणे, टूल कॉलिंग सक्षम करणे, आणि आपल्या मल्टी-एजंट अॅप्लिकेशनसाठी लाइव्ह वेब UI वायर अप करणे, अशा सर्व गोष्टी केल्या आहेत. हे सर्व पूर्णपणे आपल्या डिव्हाइसमध्ये चालत आहे.

---

## आपण काय तयार केले

| भाग | आपण काय तयार केले |
|------|------------------|
| 1 | फाउंड्री लोकल स्थापित केले, CLI द्वारे मॉडेल तपासले |
| 2 | फाउंड्री लोकल SDK API सेवा, कॅटलॉग, कॅश, आणि मॉडेल व्यवस्थापनासाठी पारंगत केले |
| 3 | SDK वापरून Python, JavaScript, आणि C# कडून OpenAI सोबत कनेक्ट केले |
| 4 | स्थानिक ज्ञान पुनर्प्राप्तीसाठी RAG पाइपलाइन तयार केली |
| 5 | व्यक्तिमत्वांसह AI एजंट्स आणि संरचित आउटपुट तयार केले |
| 6 | फीडबॅक लूपसह मल्टी-एजंट पाइपलाइन आयोजित केली |
| 7 | उत्पादनासंबंधित कॅपस्टोन अ‍ॅप्लिकेशन, झावा क्रिएटिव्ह रायटर एक्सप्लोर केले |
| 8 | एजंटसाठी मूल्यांकन-आधारित विकास कार्यप्रवाह तयार केले |
| 9 | Whisper वापरून ऑडिओ ट्रान्सक्राइब केले, डिव्हाइसवर स्पीच-टू-टेक्स्ट केले |
| 10 | ONNX Runtime GenAI सह सानुकूल Hugging Face मॉडेल कंपाईल आणि रॅन केले |
| 11 | टूल कॉलिंगसह स्थानिक मॉडेल्सना बाह्य फंक्शन्स कॉल करण्यायोग्य केले |
| 12 | झावा क्रिएटिव्ह रायटरसाठी ब्राउझर-आधारित UI रिअल-टाइम स्ट्रीमिंगसह तयार केले |

---

## पुढील कल्पना

- एजंट्ससह टूल कॉलिंग संयोजीत करून स्वायत्त कार्यप्रवाह तयार करा
- आपल्या टूल्समधून स्थानिक डेटाबेस किंवा आंतररिक REST API कॉल करा
- गुणवत्ता आणि वेग तुलना करण्यासाठी वेगवेगळे मॉडेल्स (`phi-4-mini`, `deepseek-r1-7b`) वापरून पहा
- आपल्या झावा रायटर वेब सर्व्हरसाठी प्रमाणीकरण किंवा रेट लिमिटिंग जोडा
- आपणास महत्त्व असलेल्या क्षेत्रासाठी स्वतःचे मल्टी-एजंट अ‍ॅप्लिकेशन तयार करा
- Foundry Local बदलून Azure AI Foundry वापरून क्लाउडवर तैनात करा, समान कोड, वेगळे एन्डपॉईंट
