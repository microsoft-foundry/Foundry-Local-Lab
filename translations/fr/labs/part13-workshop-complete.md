# Partie 13 : Atelier terminé

> **Félicitations !** Vous avez complété l'intégralité de l'atelier Foundry Local.

Vous êtes passé de l'installation du CLI à la création d'applications de chat, de pipelines RAG, de systèmes multi-agents, de transcription vocale, à la compilation de vos propres modèles personnalisés, à l'activation de l'appel d'outils, et à la mise en place d'une interface web en direct pour votre application multi-agent. Tout cela fonctionne entièrement sur votre appareil.

---

## Ce que vous avez construit

| Partie | Ce que vous avez construit |
|--------|----------------------------|
| 1 | Installation de Foundry Local, exploration des modèles via le CLI |
| 2 | Maîtrise de l'API SDK Foundry Local pour la gestion des services, catalogue, cache et modèles |
| 3 | Connexion depuis Python, JavaScript et C# en utilisant le SDK avec OpenAI |
| 4 | Construction d'un pipeline RAG avec récupération locale des connaissances |
| 5 | Création d'agents IA avec des personas et une sortie structurée |
| 6 | Orchestration de pipelines multi-agents avec boucles de rétroaction |
| 7 | Exploration d'une application finale en production, le Zava Creative Writer |
| 8 | Élaboration de workflows de développement basés sur l'évaluation pour les agents |
| 9 | Transcription audio avec Whisper, reconnaissance vocale locale |
| 10 | Compilation et exécution d'un modèle personnalisé Hugging Face avec ONNX Runtime GenAI |
| 11 | Activation des modèles locaux pour appeler des fonctions externes via l'appel d'outils |
| 12 | Création d'une interface utilisateur basée sur navigateur pour le Zava Creative Writer avec streaming en temps réel |

---

## Idées supplémentaires

- Combiner l'appel d'outils avec des agents pour construire des workflows autonomes
- Interroger une base de données locale ou appeler des APIs REST internes depuis vos outils
- Tester différents modèles (`phi-4-mini`, `deepseek-r1-7b`) pour comparer qualité et vitesse
- Ajouter une authentification ou une limitation de débit à votre serveur web Zava Writer
- Créer votre propre application multi-agent pour un domaine qui vous importe
- Déployer sur le cloud en remplaçant Foundry Local par Azure AI Foundry, même code, point de terminaison différent
