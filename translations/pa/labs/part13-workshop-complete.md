# ਹਿੱਸਾ 13: ਵਰਕਸ਼ਾਪ ਮੁਕੰਮਲ

> **ਬਧਾਈ ਹੋਵੇ!** ਤੁਸੀਂ ਪੂਰੀ Foundry Local ਵਰਕਸ਼ਾਪ ਨੂੰ ਮੁਕੰਮਲ ਕਰ ਲਿਆ ਹੈ।

ਤੁਸੀਂ CLI ਨੂੰ ਇੰਸਟਾਲ ਕਰਨ ਤੋਂ ਲੈ ਕੇ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ ਬਣਾਉਣ, RAG ਪਾਈਪਲਾਈਨ, ਬਹੁ-ਏਜੰਟ ਸਿਸਟਮ, ਸਪੀਚ-ਟੂ-ਟੈਕਸਟ ਟ੍ਰਾਂਸਕ੍ਰਿਪਸ਼ਨ, ਆਪਣੇ ਹੀ ਕਸਟਮ ਮਾਡਲ ਕੁੰਪਾਇਲ ਕਰਨ, ਟੂਲ ਕਾਲਿੰਗ ਯੋਗ ਬਣਾਉਣ ਅਤੇ ਆਪਣੇ ਬਹੁ-ਏਜੰਟ ਐਪਲੀਕੇਸ਼ਨ ਲਈ ਲਾਈਵ ਵੈਬ UI ਸੈੱਟ ਕਰਨ ਤੱਕ ਸਾਰਾ ਕੰਮ ਆਪਣੇ ਡਿਵਾਈਸ 'ਤੇ ਚਲਾਇਆ ਹੈ।

---

|------|----------------|

| 3 | Python, JavaScript ਅਤੇ C# ਤੋਂ SDK ਨਾਲ OpenAI ਵਰਤੀ |
| 4 | ਲੋਕਲ ਜਾਣਕਾਰੀ ਪ੍ਰਾਪਤੀ ਲਈ RAG ਪਾਈਪਲਾਈਨ ਬਣਾਈ |
| 5 | AI ਏਜੰਟ ਬਣਾਏ ਜਿਨ੍ਹਾਂ ਦੇ ਵਿਅਕਤਿਤਵ ਅਤੇ ਸ੍ਰੰਚਿਤ ਨਤੀਜੇ ਹਨ |
| 6 | ਫੀਡਬੈਕ ਲੂਪਾਂ ਨਾਲ ਬਹੁ-ਏਜੰਟ ਪਾਈਪਲਾਈਨ ਬਣਾਈ |
| 7 | ਪ੍ਰੋਡਕਸ਼ਨ ਕੈਪਸਟੋਨ ਐਪਲੀਕੇਸ਼ਨ Zava Creative Writer ਦੀ ਖੋਜ ਕੀਤੀ |
| 8 | ਏਜੰਟਾਂ ਲਈ ਮੁਲਾਂਕਣ ਅਧਾਰਿਤ ਵਿਕਾਸ ਵਰਕਫਲੋ ਬਣਾਇਆ |
| 9 | Whisper ਨਾਲ ਆਡੀਓ ਟ੍ਰਾਂਸਕ੍ਰਾਈਬ ਕੀਤਾ, ਡਿਵਾਈਸ 'ਤੇ ਸਪੀਚ-ਟੂ-ਟੈਕਸਟ |
| 10 | ONNX Runtime GenAI ਨਾਲ ਕਸਟਮ Hugging Face ਮਾਡਲ ਕੰਪਾਇਲ ਅਤੇ ਚਲਾਇਆ |
| 11 | ਟੂਲ ਕਾਲਿੰਗ ਨਾਲ ਲੋਕਲ ਮਾਡਲਾਂ ਨੂੰ ਬਾਹਰਲੇ ਫੰਕਸ਼ਨਾਂ ਨੂੰ ਕਾਲ ਕਰਨ ਦੀ ਯੋਗਤਾ ਦਿੱਤੀ |
| 12 | Zava Creative Writer ਲਈ ਬਰਾਊਜ਼ਰ-ਅਧਾਰਿਤ UI ਬਣਾਇਆ ਜਿਸ ਵਿੱਚ ਰੀਅਲ-ਟਾਈਮ ਸਟ੍ਰੀਮਿੰਗ ਹੈ |

## ਹੋਰ ਵਿਚਾਰ

- ਏਜੰਟਾਂ ਨਾਲ ਟੂਲ ਕਾਲਿੰਗ ਨੂੰ ਜੋੜਕੇ ਸੁਤੰਤਰ ਵਰਕਫਲੋ ਬਣਾਓ
- ਆਪਣੇ ਟੂਲਾਂ ਤੋਂ ਲੋਕਲ ਡੇਟਾਬੇਸ ਨਾਲ ਕੁਐਰੀ ਕਰੋ ਜਾਂ ਅੰਦਰੂਨੀ REST API ਕਾਲ ਕਰੋ
- ਵੱਖ-ਵੱਖ ਮਾਡਲਾਂ (`phi-4-mini`, `deepseek-r1-7b`) ਨੂੰ ਟੈਸਟ ਕਰਕੇ ਗੁਣਵੱਤਾ ਅਤੇ ਗਤੀ ਦੀ ਤੁਲਨਾ ਕਰੋ
- ਆਪਣੇ Zava Writer ਵੈਬ ਸਰਵਰ ਲਈ ਪ੍ਰਮਾਣਿਕਤਾ ਜਾਂ ਦਰ ਸੀਮਾ ਜੋੜੋ
- ਆਪਣੇ ਪਿਆਰੇ ਖੇਤਰ ਲਈ ਆਪਣੀ ਬਹੁ-ਏਜੰਟ ਐਪਲੀਕੇਸ਼ਨ ਬਣਾਓ
- Foundry Local ਨੂੰ Azure AI Foundry ਨਾਲ ਬਦਲ ਕੇ ਕਲਾਉਡ 'ਤੇ ਡਿਪਲੋਇ ਕਰੋ, ਇੱਕੋ ਕੋਡ ਨਾਲ, ਸਿਰਫ ਵੱਖਰਾ ਐਂਡਪੌਇੰਟ
