# ਭਾਗ 8: Foundry Local ਨਾਲ ਮੁੱਲਾਂਕਣ-ਕੇਂਦ੍ਰਤ ਵਿਕਾਸ

> **ਲਕੜੀ:** ਇੱਕ ਮੁੱਲਾਂਕਣ ਫਰੇਮਵਰਕ ਬਣਾਓ ਜੋ ਤੁਹਾਡੇ AI ਏਜੰਟਾਂ ਦੀ ਵਿਆਵਸਥਿਤ ਤੌਰ ਤੇ ਜਾਂਚ ਤੇ ਸਕੋਰ ਕਰਦਾ ਹੈ, ਇੱਕੋ ਹੀ ਸਥਾਨਕ ਮਾਡਲ ਨੂੰ ਟੈਸਟ ਹੇਠ ਏਜੰਟ ਅਤੇ ਨਿਆਂਦਾਤਾ ਦੋਵਾਂ ਲਈ ਵਰਤਦਾ ਹੈ, ਤਾਂ ਜੋ ਤੁਸੀਂ ਵਿਸ਼ਵਾਸ ਨਾਲ ਪ੍ਰਾਪਮੱਟਾਂ 'ਤੇ ਦੁਹਰਾਈ ਕਰ ਸਕੋ ਇਸ ਤੋਂ ਪਹਿਲਾਂ ਭੇਜਣ ਤੋਂ।

## ਮੁੱਲਾਂਕਣ-ਕੇਂਦ੍ਰਤ ਵਿਕਾਸ ਕਿਉਂ?

AI ਏਜੰਟ ਬਣਾਉਂਦੇ ਸਮੇਂ, "ਇਹ ਠੀਕ ਜਾਪਦਾ ਹੈ" ਕਾਫ਼ੀ ਨਹੀਂ ਹੈ। **ਮੁੱਲਾਂਕਣ-ਕੇਂਦ੍ਰਤ ਵਿਕਾਸ** ਏਜੰਟ ਆਉਟਪੁੱਟਾਂ ਨੂੰ ਕੋਡ ਵਾਂਗ ਮਾਨਦਾ ਹੈ: ਤੁਸੀਂ ਪਹਿਲਾਂ ਟੈਸਟ ਲਿਖਦੇ ਹੋ, ਗੁਣਵੱਤਾ ਨੂੰ ਮਾਪਦੇ ਹੋ, ਅਤੇ ਸਿਰਫ਼ ਜਦੋਂ ਸਕੋਰ ਕੋਈ ਸੀਮਾ ਪੂਰੀ ਕਰਦੇ ਹਨ ਤਦ ਝੱਡ ਨੂੰ ਛੱਡਦੇ ਹੋ।

ਜ਼ਾਵਾ ਕ੍ਰੀਏਟਿਵ ਰਾਈਟਰ (ਭਾਗ 7) ਵਿੱਚ, **ਸੰਪਾਦਕ ਏਜੰਟ** ਪਹਿਲਾਂ ਹੀ ਇੱਕ ਹਲਕਾ ਮੁੱਲਾਂਕਣਕਾਰ ਵਜੋਂ ਕੰਮ ਕਰਦਾ ਹੈ; ਇਹ ACCEPT ਜਾਂ REVISE ਦਾ ਫੈਸਲਾ ਕਰਦਾ ਹੈ। ਇਹ ਲੈਬ ਇਸ ਪੈਟਰਨ ਨੂੰ ਇੱਕ ਦੁਹਰਾਏ ਜਾਣ ਯੋਗ ਮੁੱਲਾਂਕਣ ਫਰੇਮਵਰਕ ਵਿੱਚ ਬਦਲਦਾ ਹੈ ਜੋ ਤੁਸੀਂ ਕਿਸੇ ਵੀ ਏਜੰਟ ਜਾਂ ਪਾਈਪਲਾਈਨ 'ਤੇ ਲਾਗੂ ਕਰ ਸਕਦੇ ਹੋ।

| ਸਮੱਸਿਆ | ਹੱਲ |
|---------|----------|
| ਪ੍ਰਾਂਪਟ ਬਦਲਾਅ ਗੁਣਵੱਤਾ ਨੂੰ ਚੁਪਚਾਪ ਤੋੜਦੇ ਹਨ | **ਸੋਨੇ ਦਾ ਡੈਟਾਸੈੱਟ** ਰਿਗਰੇਸ਼ਨ ਫੜਦਾ ਹੈ |
| "ਇੱਕ ਉਦਾਹਰਨ 'ਤੇ ਕੰਮ ਕਰਦਾ ਹੈ" ਭ੍ਰਮ | **ਕਈ ਟੈਸਟ ਕੇਸ** ਕਿਨਾਰੇ ਦੇ ਮਾਮਲੇ ਵਿਖਾਉਂਦੇ ਹਨ |
| ਵਿਆਖਿਆਤਮਕ ਗੁਣਵੱਤਾ ਮੁੱਲਾਂਕਣ | **ਨਿਯਮ-ਆਧਾਰਿਤ + LLM-ਨਿਆਂਦਾਤਾ ਸਕੋਰਿੰਗ** ਨੰਬਰ ਪੇਸ਼ ਕਰਦਾ ਹੈ |
| ਪ੍ਰਾਂਪਟ ਵੈਰੀਐਂਟ ਤੁਲਨਾ ਕਰਨ ਦਾ ਕੋਈ ਤਰੀਕਾ ਨਹੀਂ | **ਬਾਜੂ-ਬਾਜੂ ਮੁੱਲਾਂਕਣ ਦੌੜਾਂ** ਨਾਲ ਕੁੱਲ ਸਕੋਰ |

## ਮੁੱਖ ਸਿਧਾਂਤ

### 1. ਸੋਨੇ ਦਾ ਡੈਟਾਸੈੱਟ

ਇੱਕ **ਸੋਨੇ ਦਾ ਡੈਟਾਸੈੱਟ** ਚੁਣੇ ਹੋਏ ਟੈਸਟ ਕੇਸਾਂ ਦਾ ਇੱਕ ਸੈੱਟ ਹੈ ਜਿਸ ਵਿੱਚ ਜਾਣੇ-ਪਛਾਣੇ ਉਮੀਦਵਾਰ ਨਤੀਜੇ ਹੁੰਦੇ ਹਨ। ਹਰ ਟੈਸਟ ਕੇਸ ਵਿੱਚ ਸ਼ਾਮਲ ਹੈ:

- **ਇਨਪੁਟ**: ਪ੍ਰਾਂਪਟ ਜਾਂ ਪ੍ਰਸ਼ਨ ਜੋ ਏਜੰਟ ਨੂੰ ਭੇਜਣਾ ਹੈ
- **ਉਮੀਦਵਾਰ ਨਤੀਜਾ**: ਇੱਕ ਸਹੀ ਜਾਂ ਉੱਚ-ਗੁਣਵੱਤਾ ਵਾਲਾ ਜਵਾਬ ਕੀ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ (ਕੀਵਰਡ, ਬਣਤਰ, ਤੱਥ)
- **ਸ਼੍ਰੇਣੀ**: ਰਿਪੋਰਟਿੰਗ ਲਈ ਗਰੁੱਪਿੰਗ (ਜਿਵੇਂ "ਤੱਥਾ ਸਹੀ", "ਟੋਨ", "ਪੂਰਨਤਾ")

### 2. ਨਿਯਮ-ਆਧਾਰਿਤ ਜਾਂਚਾਂ

ਤੇਜ਼, ਨਿਸ਼ਚਿਤ ਜਾਂਚਾਂ ਜੋ LLM ਦੀ ਲੋੜ ਨਹੀਂ ਹਨ:

| ਜਾਂਚ | ਕੀ ਟੈਸਟ ਕਰਦਾ ਹੈ |
|-------|--------------|
| **ਲੰਬਾਈ ਸੀਮਾਵਾਂ** | ਜਵਾਬ ਨਾ ਦੇਰ ਛੋਟਾ (ਅਲਸੀ) ਜਾਂ ਨਾ ਬਹੁਤ ਲੰਬਾ (ਬੇਲੜਾਉ) ਹੋਵੇ |
| **ਲੋੜੀਂਦੇ ਕੀਵਰਡ** | ਜਵਾਬ ਵਿੱਚ ਉਮੀਦਵਾਰ ਸ਼ਬਦ ਜਾਂ ਇਕਾਈਆਂ ਹੋਣ |
| **ਫਾਰਮੈਟ ਪ੍ਰਮਾਣਿਕਤਾ** | JSON ਪਾਰਸ ਕਰਨਯੋਗ, ਮਾਰਕਡਾਊਨ ਹੈਡਰ ਮੌਜੂਦ |
| **ਨিষিদ্ধ ਸਮੱਗਰੀ** | ਕੋਈ ਗਲਤ ਬ੍ਰਾਂਡ ਨਾਮ, ਕੋਈ ਮੁਕਾਬਲੇਦਾਰ ਜ਼ਿਕਰ ਨਹੀਂ |

### 3. LLM-ਨਿਆਂਦਾਤਾ

ਆਪਣੇ ਹੀ ਆਉਟਪੁੱਟ (ਜਾਂ ਵੱਖਰੇ ਪ੍ਰਾਂਪਟ ਵੈਰੀਐਂਟ ਤੋਂ ਆਉਟਪੁੱਟ) ਨੂੰ ਗਰੇਡ ਕਰਨ ਲਈ ਇੱਕੋ ਹੀ ਸਥਾਨਕ ਮਾਡਲ ਨੂੰ ਵਰਤੋ। ਨਿਆਂਦਾਤਾ ਨੂੰ ਮਿਲਦਾ ਹੈ:

- ਮੂਲ ਪ੍ਰਸ਼ਨ
- ਏਜੰਟ ਦਾ ਜਵਾਬ
- ਗਰੇਡਿੰਗ ਮਾਪਦੰਡ

ਅਤੇ ਵਾਪਸ ਕਰਦਾ ਹੈ ਇੱਕ ਸੰਰਚਿਤ ਸਕੋਰ। ਇਹ ਭਾਗ 7 ਤੋਂ ਸੰਪਾਦਕ ਪੈਟਰਨ ਦਾ ਆਗਾਂਹ ਹੈ ਪਰ ਇਕ ਟੈਸਟ ਸੁੱਟ 'ਚ ਵਿਵਸਥਿਤ ਤੌਰ ਤੇ ਲਾਗੂ ਕੀਤਾ ਗਿਆ।

### ਅਭਿਆਸ 2 - ਸੋਨੇ ਦੇ ਡੈਟਾਸੈੱਟ ਨੂੰ ਸਮਝੋ

ਮੁੱਲਾਂਕਣ ਨਮੂਨੇ ਵਿੱਚ ਪਰਿਭਾਸ਼ਿਤ ਟੈਸਟ ਕੇਸ ਦੇਖੋ। ਹਰ ਟੈਸਟ ਕੇਸ ਵਿੱਚ ਸ਼ਾਮਲ ਹੈ:

```
{
  "input":    "What tools do I need to build a deck?",
  "expected": ["saw", "drill", "screws", "level"],
  "category": "product-recommendation"
}
```

**ਵਿਚਾਰ ਕਰਨ ਦੇ ਪ੍ਰਸ਼ਨ:**
1. ਉਮੀਦਵਾਰ ਮੁੱਲ **ਕੀਵਰਡ** ਕਿਉਂ ਹਨ ਪੂਰੇ ਵਾਕਾਂ ਦੀ ਥਾਂ?
2. ਭਰੋਸੇਮੰਦ ਮੁੱਲਾਂਕਣ ਲਈ ਤੁਹਾਨੂੰ ਕਿੰਨੇ ਟੈਸਟ ਕੇਸ ਚਾਹੀਦੇ ਹਨ?
3. ਆਪਣੇ ਐਪਲੀਕੇਸ਼ਨ ਲਈ ਤੁਸੀਂ ਕਿਹੜੀਆਂ ਸ਼੍ਰੇਣੀਆਂ ਸ਼ਾਮਲ ਕਰੋਗੇ?

### ਅਭਿਆਸ 3 - ਨਿਯਮ-ਆਧਾਰਿਤ ਅਤੇ LLM-ਨਿਆਂਦਾਤਾ ਸਕੋਰਿੰਗ ਨੂੰ ਸਮਝੋ

ਮੁੱਲਾਂਕਣ ਫਰੇਮਵਰਕ **ਦੋ ਸਕੋਰਿੰਗ ਪੱਧਰ** ਵਰਤਦਾ ਹੈ:

#### ਨਿਯਮ-ਆਧਾਰਿਤ ਜਾਂਚਾਂ (ਤੇਜ਼, ਨਿਸ਼ਚਿਤ)

```
✓ Length: 50-500 words       → 1 point
✓ Keywords: 3/4 found        → 0.75 points  
✗ Forbidden: mentions "Home Depot" → 0 points
─────────────────────────────
Rule score: 0.58 / 1.0
```

#### LLM-ਨਿਆਂਦਾਤਾ (ਸੂਖਮ, ਗੁਣਾਤਮਕ)

ਇੱਕੋ ਸਥਾਨਕ ਮਾਡਲ ਨਿਆਂਦਾਤਾ ਵਜੋਂ ਕੰਮ ਕਰਦਾ ਹੈ ਇੱਕ ਸੰਰਚਿਤ ਰੂਬ੍ਰਿਕ ਨਾਲ:

```
Rate this response on a scale of 1-5:
- 1: Completely wrong or irrelevant
- 2: Partially correct but missing key information
- 3: Adequate but could be improved
- 4: Good response with minor issues
- 5: Excellent, comprehensive, well-structured

Score: 4
Reasoning: The response correctly identifies all necessary tools
and provides practical advice, but could include safety equipment.
```

**ਵਿਚਾਰ ਕਰਨ ਦੇ ਪ੍ਰਸ਼ਨ:**
1. ਤੁਸੀਂ ਕਦੋਂ ਨਿਯਮ-ਆਧਾਰਿਤ ਜਾਂਚਾਂ 'ਤੇ LLM-ਨਿਆਂਦਾਤਾ ਨਾਲ ਭਰੋਸਾ ਕਰੋਗੇ?
2. ਕੀ ਇੱਕ ਮਾਡਲ ਆਪਣੇ ਹੀ ਨਤੀਜੇ ਨੂੰ ਭਰੋਸੇਯੋਗ ਤੌਰ ਤੇ ਨਿਆਂ ਕਰ ਸਕਦਾ ਹੈ? ਸੀਮਾਵਾਂ ਕੀ ਹਨ?
3. ਇਹ ਭਾਗ 7 ਦੇ ਸੰਪਾਦਕ ਏਜੰਟ ਪੈਟਰਨ ਨਾਲ ਕਿਵੇਂ ਤੁਲਨਾ ਕਰਦਾ ਹੈ?

### ਅਭਿਆਸ 4 - ਪ੍ਰਾਂਪਟ ਵੈਰੀਐਂਟਾਂ ਦੀ ਤੁਲਨਾ ਕਰੋ

ਨਮੂਨਾ **ਦੋ ਪ੍ਰਾਂਪਟ ਵੈਰੀਐਂਟ** ਨੂੰ ਉਹੀ ਟੈਸਟ ਕੇਸਾਂ 'ਤੇ ਦੌੜਾਉਂਦਾ ਹੈ:

| ਵੈਰੀਐਂਟ | ਪ੍ਰਣਾਲੀ ਪ੍ਰਾਂਪਟ ਸ਼ੈਲੀ |
|---------|-------------------|
| **ਬੇਸਲਾਈਨ** | ਸਧਾਰਣ: "ਤੁਸੀਂ ਇੱਕ ਮਦਦਗਾਰ ਸਹਾਇਕ ਹੋ" |
| **ਖਾਸ** | ਵਿਸਥਾਰਪੂਰਕ: "ਤੁਸੀਂ ਇੱਕ ਜ਼ਾਵਾ DIY ਮਾਹਿਰ ਹੋ ਜੋ ਵਿਸ਼ੇਸ਼ ਉਤਪਾਦ ਸਿਫਾਰਸ਼ ਕਰਦਾ ਹੈ ਅਤੇ ਕਦਮ-ਦਰ-ਕਦਮ ਮਦਦ ਦਿੰਦਾ ਹੈ" |

ਚਲਾਉਣ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ ਇਸ ਤਰ੍ਹਾਂ ਦਾ ਸਕੋਰਕਾਰਡ ਵੇਖੋਗੇ:


2. ਖਾਸ ਪ੍ਰਾਂਪਟ ਨੂੰ ਹੋਰ ਨਿਸ਼ਚਿਤ ਬਣਾਓ। ਕੀ ਸਕੋਰ ਬਿਹਤਰ ਹੁੰਦਾ ਹੈ?  
3. ਤੀਜਾ ਪ੍ਰਾਂਪਟ ਵੈਰੀਐਂਟ ਸ਼ਾਮਲ ਕਰੋ ਅਤੇ ਤਿੰਨਾਂ ਦੀ ਤੁਲਨਾ ਕਰੋ।

### ਅਭਿਆਸ 5 - ਆਪਣੇ ਸਵੌਂ ਏਜੰਟ 'ਤੇ ਮੁੱਲਾਂਕਣ ਲਾਗੂ ਕਰੋ

ਮੁੱਲਾਂਕਣ ਫਰੇਮਵਰਕ ਨੂੰ ਆਪਣੇ ਏਜੰਟਾਂ ਲਈ ਟੈਂਪਲੇਟ ਵਜੋਂ ਵਰਤੋ:

1. **ਆਪਣਾ ਸੋਨੇ ਦਾ ਡੈਟਾਸੈੱਟ ਪਰਿਭਾਸ਼ਿਤ ਕਰੋ**: 5 ਤੋਂ 10 ਟੈਸਟ ਕੇਸ ਲਿਖੋ ਜਿਨ੍ਹਾਂ ਵਿੱਚ ਉਮੀਦਵਾਰ ਕੀਵਰਡ ਹੋਣ।  
2. **ਆਪਣਾ ਪ੍ਰਣਾਲੀ ਪ੍ਰਾਂਪਟ ਲਿਖੋ**: ਉਹ ਏਜੰਟ ਹੁਕਮ ਜੋ ਤੁਸੀਂ ਟੈਸਟ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ।  
3. **ਮੁੱਲਾਂਕਣ ਚਲਾਓ**: ਬੇਸਲਾਈਨ ਸਕੋਰ ਪ੍ਰਾਪਤ ਕਰੋ।  
4. **ਇਤਰਾਜ਼ ਕਰੋ**: ਪ੍ਰਾਂਪਟ 'ਚ ਸੋਧ ਕਰੋ, ਮੁੜ ਚਲਾਓ, ਅਤੇ ਤੁਲਨਾਤਮਕ ਜाँच ਕਰੋ।  
5. **ਸীমਾ ਨਿਰਧਾਰਿਤ ਕਰੋ**: ਜਿਵੇਂ "0.75 ਮਿਲ ਕੇ ਸਕੋਰ ਤੋਂ ਹੇਠਾਂ ਨਾ ਭੇਜੋ"।

### ਅਭਿਆਸ 6 - ਜ਼ਾਵਾ ਸੰਪਾਦਕ ਪੈਟਰਨ ਨਾਲ ਸੰਪਰਕ

ਜ਼ਾਵਾ ਕ੍ਰੀਏਟਿਵ ਰਾਈਟਰ ਦੇ ਸੰਪਾਦਕ ਏਜੰਟ ਨੂੰ ਵੇਖੋ (`zava-creative-writer-local/src/api/agents/editor/editor.py`):

```python
# ਸੰਪਾਦਕ ਉਤਪਾਦਨ ਵਿੱਚ ਇੱਕ ਐਲਐਲਐਮ-ਕੋਰਟੀ ਦੀ ਭੂਮਿਕਾ ਨਿਭਾ ਰਿਹਾ ਹੈ:
{"decision": "accept/revise", "editorFeedback": "...", "researchFeedback": "..."}
```

ਇਹ ਭਾਗ 8 ਦੇ LLM-ਨਿਆਂਦਾਤਾ ਨਾਲ **ਉਹੀ ਧਾਰਨਾ** ਹੈ, ਪਰ ਉਤਪਾਦਨ ਪਾਈਪਲਾਈਨ ਵਿੱਚ ਸਿੱਧੀ ਤਰ੍ਹਾਂ ਲਾਗੂ ਕੀਤਾ ਗਿਆ ਹੈ, ਬਾਹਰਲੇ ਟੈਸਟ ਸੁੱਟ ਵਿੱਚ ਨਹੀਂ। ਦੋਹਾਂ ਪੈਟਰਨ:

- ਮਾਡਲ ਤੋਂ ਸੰਰਚਿਤ JSON ਨਤੀਜਾ ਲੈਂਦੇ ਹਨ।  
- ਪ੍ਰਣਾਲੀ ਪ੍ਰਾਂਪਟ ਵਿੱਚ ਪਰਿਭਾਸ਼ਿਤ ਗੁਣਵੱਤਾ ਮਾਪਦੰਡ ਲਗਾਉਂਦੇ ਹਨ।  
- ਪਾਸ/ਫੇਲ ਫੈਸਲਾ ਕਰਦੇ ਹਨ।  

**ਫਰਕ:** ਸੰਪਾਦਕ ਉਤਪਾਦਨ (ਹਰ ਰਿਕਵੇਸਟ 'ਤੇ) ਚਲਦਾ ਹੈ। ਮੁੱਲਾਂਕਣ ਫਰੇਮਵਰਕ ਵਿਕਾਸ ਦੌਰਾਨ (ਤੁਹਾਡੇ ਭੇਜਣ ਤੋਂ ਪਹਿਲਾਂ) ਚਲਦਾ ਹੈ।
