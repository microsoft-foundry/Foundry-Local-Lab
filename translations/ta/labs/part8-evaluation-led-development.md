# பகுதி 8: Foundry Local உடன் மதிப்பீடு-மேற்கொண்ட வளர்ச்சி

> **இலக்கு:** உங்கள் AI முகவர்களை முறையாக சோதிக்கும் மற்றும் மதிப்பீடு செய்யும் மதிப்பீட்டு வடிவமைப்பை உருவாக்குதல், கடுமையான சோதனைக்காக மற்றும் குற்றங்கள் இல்லாமல் நம்பிக்கையுடன் தனிவிதானங்களை சோதிப்பதற்குப் பயன்படுத்தப்படும் அதே உள்ளூர் மாதிரியை முகவராகவும் மேடயம் (judge) ஆகவும் பயன்படுத்துவது.

## ஏன் மதிப்பீடு-மேற்கொண்ட வளர்ச்சி?

AI முகவர்களை உருவாக்கும்போது, "உண்மையில் சரியாக தெரிகிறது" என்பது போதுமானதாக இல்லாது. **மதிப்பீடு-மேற்கொண்ட வளர்ச்சி** முகவரின் வெளியீடுகளை குறியீடு போல பார்த்துக்கொள்கிறது: முதலில் சோதனைகளை எழுதுங்கள், தரத்தை அளவிடுங்கள் மற்றும் மதிப்பெண்கள் ஒரு அதிகபட்ச அளவுக்கு சென்றால் மட்டுமே வெளியிடுங்கள்.

Zava Creative Writer (பகுதி 7) இல், **திருத்துநர் முகவர்** ஏற்கனவே எளிய மதிப்பீட்டாளராக நடிக்கிறது; இது ACCEPT அல்லது REVISE முடிவுகளை எடுக்கிறது. இந்த ஆய்வு இதை ஒரு மீண்டும் பயன்படும் மதிப்பீட்டு வடிவமைப்பாகும், அதை எந்த முகவருக்கும் அல்லது குழாய்க்குமெல்லாம் பயன்படுத்தலாம்.

| பிரச்சினை | தீர்வு |
|---------|----------|
| தனிவிதான மாற்றங்கள் சத்தமின்றி தரத்தை பாதிக்கின்றன | **தங்க dataset** திரும்பிச் செல்லும் பிழைகளை பிடிக்கிறது |
| "ஒரு உதாரணத்தில் வேலை செய்கிறது" எனும் பாகுபாடு | **பல சோதனை வழக்குகள்** கடைசிப் பகுதிகளை வெளிப்படுத்துகின்றன |
| பொருத்தாத தர மதிப்பீடு | **விதி அடிப்படையிலான + LLM-ஆக-மேடயம் மதிப்பீடு** எண்களை வழங்குகிறது |
| தனிவிதான வகைகளை ஒப்பிட வழி இல்லை | **பக்கපக்கமாக மதிப்பீடு இயக்கங்கள்** ஒன்றிணைந்த மதிப்பெண்களுடன் |

## முக்கியக் கருத்துக்கள்

### 1. தங்க தரவுத்தொகுப்புகள்

**தங்க dataset** என்பது தெரிந்த எதிர்பார்க்கப்பட்ட வெளியீடுகளுடன் கூடிய தேர்ந்தெடுக்கப்பட்ட சோதனை வழக்குகளின் தொகுப்பு. ஒவ்வொரு சோதனை வழக்கிலும்:

- **உள்ளீடு**: முகவருக்கு அனுப்ப வேண்டிய தனிவிதானம் அல்லது கேள்வி
- **எதிர்பார்க்கப்படும் வெளியீடு**: சரியான அல்லது உயர் தர பதிலில் இருக்க வேண்டியது (முக்கிய சொற்கள், கட்டமைப்பு, உண்மைகள்)
- **வகுப்பு**: அறிக்கைக்கான குழுக்கள் (எ.கா. "உண்மைத் தன்மை", "தொனிம்", "கூர்மை")

### 2. விதி அடிப்படையிலான சோதனைகள்

LLM தேவையில்லாத விரைவு, நியாயமான சோதனைகள்:

| சோதனை | அது சோதிக்கிறது |
|-------|--------------|
| **நீளம் வரம்புகள்** | பதில் குறைவு (லேசி) அல்லது நீண்ட (ஹிறிறு) ஆக இருக்கக்கூடாது |
| **கேட்கப்படும் முக்கிய சொற்கள்** | பதில் எதிர்பார்க்கப்படும் சொற்கள் அல்லது உருப்படியைக் குறிப்பிடுகிறது |
| **வடிவமைப்பு சரிபார்ப்பு** | JSON பிரிப்பு சாத்தியம், Markdown தலைப்புகள் உள்ளன |
| **தடைபட்ட உள்ளடக்கம்** | புனைவு வர்த்தக பெயர்கள் இல்லை, போட்டியாளர்கள் குறிப்புகள் இல்லை |

### 3. LLM-ஆக-மேடயம்

உருவாக்கப்படும் தனி மாதிரி வெளியீடுகளை (அல்லது வேறு தனிவிதானம்) மதிப்பெண்கள் வைப்பதற்கு **அதே உள்ளூர் மாதிரியை**ப் பயன்படுத்துங்கள். மேடயம் பெறுகிறது:

- அசல் கேள்வி
- முகவரின் பதில்
- மதிப்பீட்டு குறிப்புகள்

மற்றுமொரு அமைப்பில் மதிப்பெண்களை வழங்கி திரும்ப அனுப்புகிறது. இது பகுதிக 7 இல் இருந்து திருத்துநர் மாதிரியை சுருக்கமாகப் பயன்படுத்துகிறது, ஆனால் பரபரப்பான சோதனை தொகுப்பில் முறையாக செயல்படுத்தப்பட்டுள்ளது.

### பயிற்சி 2 - தங்க Dataset ஐ புரிந்துகொள்

மதிப்பீட்டு மாதிரியில் வகைப்படுத்தப்பட்ட சோதனை வழக்குகளை கவனித்து ஆய்வு செய்க. ஒவ்வொரு வழக்கிலும்:

```
{
  "input":    "What tools do I need to build a deck?",
  "expected": ["saw", "drill", "screws", "level"],
  "category": "product-recommendation"
}
```

**அறிந்துகொள்ள கேள்விகள்:**
1. எதிர்பார்க்கப்படும் மதிப்புகள் முழு வாக்கியங்களாக இல்லாமல் **முக்கிய சொற்களாக** இருக்க காரணம் என்ன?
2. நம்பகமான மதிப்பீட்டிற்கு எத்தனை சோதனை வழக்குகள் தேவை?
3. உங்கள் சொந்த பயன்பாட்டுக்காக எந்த வகுக்களை நீங்கள் சேர்க்க விரும்புவீர்கள்?

### பயிற்சி 3 - விதி அடிப்படையிலான மற்றும் LLM-ஆக-மேடயம் மதிப்பீட்டை புரிந்து கொள்

மதிப்பீட்டு வடிவமைப்பு **இரு மதிப்பீட்டு அடுக்கு** பயன்படுத்துகிறது:

#### விதி அடிப்படையிலான சோதனைகள் (விரைவு, நியாயமானவை)

```
✓ Length: 50-500 words       → 1 point
✓ Keywords: 3/4 found        → 0.75 points  
✗ Forbidden: mentions "Home Depot" → 0 points
─────────────────────────────
Rule score: 0.58 / 1.0
```

#### LLM-ஆக-மேடயம் (மிகுந்த விரிவான, தரமான)

அதே உள்ளூர் மாதிரி கட்டமைக்கப்பட்ட மதிப்பெண் அட்டவணையுடன் மேடயம் ஆக செயல்படுகிறது:

```
Rate this response on a scale of 1-5:
- 1: Completely wrong or irrelevant
- 2: Partially correct but missing key information
- 3: Adequate but could be improved
- 4: Good response with minor issues
- 5: Excellent, comprehensive, well-structured

Score: 4
Reasoning: The response correctly identifies all necessary tools
and provides practical advice, but could include safety equipment.
```

**அறிந்துகொள்ள கேள்விகள்:**
1. எந்த நேரத்தில் விதி அடிப்படையிலான சோதனைகளை LLM-ஆக-மேடயத்துக்கு மாறாக நம்புவீர்கள்?
2. ஒரு மாதிரி தன் வெளியீடுகளை நம்பகமாக மதிப்பீடு செய்யக்கூடுமா? அதற்குள்ள வரம்புகள் என்ன?
3. இது பகுதி 7 இல் இருந்த திருத்துநர் முகவர் மாதிரியை எப்படி ஒப்பிடுகிறது?

### பயிற்சி 4 - தனிவிதானங்களை ஒப்பிடு

மாதிரி **இரு தனிவிதானங்களை** அதே சோதனை வழக்குகளை எதிர்த்து இயக்குகிறது:

| தனிவிதானம் | அமைப்பு தனிவிதான சுயம் |
|---------|-------------------|
| **அடிப்படை** | பொதுவான: "நீங்கள் உதவியாளர்" |
| **திறம்பட அமைக்கப்பட்டது** | விரிவான: "நீங்கள் Zava DIY நிபுணர்; குறிப்பிட்ட தயாரிப்புகளை பரிந்துரைப்பீர்கள் மற்றும் படித்தடிகளான வழிகாட்டுதலை வழங்குவீர்கள்" |

இயக்கும் பிறகு, நீங்கள் கீழ்க்கண்ட மாதிரி ஒரு மதிப்பெண் அட்டவணையை பார்க்க முடியும்:

```
╔══════════════════════════════════════════════════════════════╗
║                    EVALUATION SCORECARD                      ║
╠══════════════════════════════════════════════════════════════╣
║ Prompt Variant    │ Rule Score │ LLM Score │ Combined       ║
╠═══════════════════╪════════════╪═══════════╪════════════════╣
║ Baseline          │ 0.62       │ 3.2 / 5   │ 0.62           ║
║ Specialised       │ 0.81       │ 4.1 / 5   │ 0.81           ║
╚══════════════════════════════════════════════════════════════╝
```

**பயிற்சிகள்:**
1. மதிப்பீட்டை இயக்கி ஒவ்வொரு தனிவிதான மதிப்பெண்களையும் பதிவு செய்க
2. திறம்பட அமைக்கப்பட்ட தனிவிதானத்தை மேலும் துல்லியமாக மாற்றுங்கள். மதிப்பெண் மேம்படுமா?
3. மூன்றாவது தனிவிதானத்தை சேர்த்து மூன்றையும் ஒப்பிடு.
4. மாதிரி அலையாஸ் (எ.கா. `phi-4-mini` vs `phi-3.5-mini`) மாற்றி முடிவுகளை ஒப்பிடுங்கள்.

### பயிற்சி 5 - உங்கள் சொந்த முகவருக்கான மதிப்பீடு பயன்படுத்துங்கள்

உங்கள் சொந்த முகவர்களுக்கான மாதிரியாக மதிப்பீட்டு வடிவமைப்பைப் பயன்படுத்துங்கள்:

1. **உங்கள் தங்க dataset ஐ வரையறுக்கவும்**: எதிர்பார்க்கப்படும் முக்கிய வார்த்தைகளுடன் 5-10 சோதனை வழக்குகளை எழுது.
2. **உங்கள் அமைப்பு தனிவிதானத்தை எழுதுங்கள்**: நீங்கள் சோதிக்க விரும்பும் முகவர் அறிவுறுத்தல்கள்.
3. **மதிப்பீட்டை இயக்குங்கள்:** அடிப்படை மதிப்பெண்களைப் பெறுங்கள்.
4. **மீண்டும் முயற்சி செய்யுங்கள்:** தனிவிதானத்தை திருத்தி மீண்டும் இயக்கி ஒப்பிடுங்கள்.
5. **ஒரு குறுக்கீட்டை அமைக்கவும்:** உதா., "0.75 கூட்டுத் தரத்தை கீழ் விட வேண்டாம்" எனக் குறிக்கவும்.

### பயிற்சி 6 - Zava திருத்துநர் மாதிரிக்கு இணைப்பு

Zava Creative Writer இன் திருத்துநர் முகவரைக் கவனியுங்கள் (`zava-creative-writer-local/src/api/agents/editor/editor.py`):

```python
# ஆசிரியர் ஒரு உற்பத்தி நிலை LLM-ஆல் நீதிபதியாக செயல்படுகிறார்:
{"decision": "accept/revise", "editorFeedback": "...", "researchFeedback": "..."}
```

இது **பகுதி 8 இன் LLM-ஆக-மேடயம்** யோசனைக்கு இணையானது, ஆனாலும் இது அப்புறப்படுத்தப்பட்ட சோதனை தொகுப்பின் முன்புறத்தில் அல்லாமல் தயாரிப்பு குழாயில் புதுப்பிக்கப்பட்டுள்ளது. இரண்டுமே:

- மாதிரியில் இருந்து கட்டமைக்கப்பட்ட JSON வெளியீட்டை பயன்படுத்துகின்றன.
- அமைப்பு தனிவிதானத்தில் வரையறுக்கப்பட்ட தரக் குறிப்புகளைப் பயன்படுத்துகின்றன.
- கடந்து/தவிர்க்க முடிவெடுக்கின்றன.

**வேறுபாடு:** திருத்துநர் தயாரிப்பில் (ஒவ்வொரு கோரிக்கைக்கும்) இயங்கும். மதிப்பீட்டு வடிவமைப்பு வளர்ச்சியில் (வழங்கும் முன்பு) இயங்கும்.

|---------|----------|
| **தங்க datasets** | சோதனை வழக்குகளை ஆரம்ப கட்டத்தில் தேர்ந்தெடுங்கள்; அவை உங்கள் AI க்கான பின்வாங்கல் சோதனைகள் |
| **விதி அடிப்படையிலான சோதனைகள்** | விரைவு, நியாயமான, தெளிவான தோல்விகளை பிடிக்கிறது (நீளம், முக்கிய சொற்கள், வடிவமைப்பு) |
| **LLM-ஆக-மேடயம்** | அதே உள்ளூர் மாதிரியால் விரிவுபடுத்தப்பட்ட தர மதிப்பீடு |
| **தனிவிதான ஒப்பீடு** | அதே சோதனைத் தொகுப்பில் பல தனிவிதானங்களை இயக்கு சிறந்ததை தேர்வு செய் |
| **கருவியில் உள்ள நன்மை** | அனைத்து மதிப்பீடுகளும் உள்ளூரில் இயங்கும்: API கட்டணங்கள் இல்லை, விகித வரம்புகள் இல்லை, தரவு உங்கள் கணக்கிலிருந்து வெளியே செல்லாது |
| **வழங்குவதற்கு முன் மதிப்பீடு** | தர நிலைகளை அமைத்து மதிப்பீட்டு மதிப்பெண்களின் அடிப்படையில் வெளியீடுகளை கட்டுப்படு |

## அடுத்த படிகள்

- **அளவை மேம்படுத்துக**: உங்கள் தங்க dataset இல் அதிக சோதனை வழக்குகளையும் வகைகளையும் சேர்க்கவும்.
- **தானியங்கி செய்க**: மதிப்பீடுகளை உங்கள் CI/CD குழாயில் ஒருங்கிணைக்கவும்.
- **மேம்பட்ட மேடயர்கள்**: சோதிக்கும் சிறிய மாதிரியின் வெளியீடுகளை மதிப்பிட பெரிய மாதிரியை மேடையாகப் பயன்படுத்தவும்.
- **நேரத்தால் கண்காணிக்கவும்**: மதிப்பீட்டு முடிவுகளை சேமித்து தனிவிதானங்கள் மற்றும் மாதிரி பதிப்புகளுக்கு இடையில் ஒப்பிடவும்.
