# भाग 13: कार्यशाला पूरी हुई

> **बधाई हो!** आपने पूरी Foundry Local कार्यशाला पूरी कर ली है।

आपने CLI इंस्टॉल करने से लेकर चैट एप्लिकेशन बनाने, RAG पाइपलाइनों, मल्टी-एजेंट सिस्टम्स, स्पीच-टू-टेक्स्ट ट्रांसक्रिप्शन, अपने कस्टम मॉडल्स को कंपाइल करने, टूल कॉलिंग सक्षम करने, और अपने मल्टी-एजेंट एप्लिकेशन के लिए लाइव वेब UI कनेक्ट करने तक का सफर पूरा किया। यह सब कुछ पूरी तरह से आपके डिवाइस पर चलता है।

---

## आपने क्या बनाया

| भाग | आपने क्या बनाया |
|------|----------------|
| 1 | Foundry Local इंस्टॉल किया, CLI के माध्यम से मॉडल्स का अन्वेषण किया |
| 2 | Foundry Local SDK API में महारत हासिल की सेवा, कैटलॉग, कैश, और मॉडल प्रबंधन के लिए |
| 3 | SDK का उपयोग करके Python, JavaScript, और C# से OpenAI से जुड़ा |
| 4 | स्थानीय ज्ञान पुनःप्राप्ति के साथ एक RAG पाइपलाइन बनाई |
| 5 | पर्सोनास और स्ट्रक्चर्ड आउटपुट के साथ AI एजेंट बनाए |
| 6 | फीडबैक लूप्स के साथ मल्टी-एजेंट पाइपलाइनों का संचालन किया |
| 7 | प्रोडक्शन कैपस्टोन एप्लिकेशन, Zava Creative Writer का अन्वेषण किया |
| 8 | एजेंट्स के लिए मूल्यांकन-आधारित विकास वर्कफ़्लोज़ बनाए |
| 9 | Whisper के साथ ऑडियो ट्रांसक्राइब किया, डिवाइस पर स्पीच-टू-टेक्स्ट |
| 10 | ONNX Runtime GenAI के साथ एक कस्टम Hugging Face मॉडल को कंपाइल और चलाया |
| 11 | टूल कॉलिंग के साथ बाहरी फ़ंक्शंस को कॉल करने के लिए स्थानीय मॉडल सक्षम किए |
| 12 | Zava Creative Writer के लिए रीयल-टाइम स्ट्रीमिंग के साथ ब्राउज़र-आधारित UI बनाया |

---

## आगे के विचार

- टूल कॉलिंग को एजेंट्स के साथ जोड़कर स्वायत्त वर्कफ़्लोज़ बनाएं
- अपने टूल्स से स्थानीय डेटाबेस क्वेरी करें या आंतरिक REST APIs कॉल करें
- गुणवत्ता और गति की तुलना के लिए विभिन्न मॉडल (`phi-4-mini`, `deepseek-r1-7b`) आज़माएं
- अपनी Zava Writer वेब सर्वर में प्रमाणीकरण या दर सीमांकन जोड़ें
- अपने पसंदीदा डोमेन के लिए स्वयं का मल्टी-एजेंट एप्लिकेशन बनाएं
- Foundry Local को Azure AI Foundry के साथ बदलकर क्लाउड पर तैनात करें, वही कोड, अलग एंडपॉइंट
