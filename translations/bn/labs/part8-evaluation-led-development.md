# অংশ ৮: Foundry Local এর মাধ্যমে মূল্যায়ন-নিয়ন্ত্রিত উন্নয়ন

> **লক্ষ্য:** একটি মূল্যায়ন কাঠামো তৈরি করা যা আপনার AI এজেন্টদের সিস্টেম্যাটিকভাবে পরীক্ষা এবং স্কোর করবে, একই লোকাল মডেলকে পরীক্ষাধীন এজেন্ট ও বিচারক উভয় হিসাবেই ব্যবহার করবে, যাতে আপনি আত্মবিশ্বাসের সাথে প্রম্পট পরিবর্তন করতে পারেন পূর্বে শিপিং করার আগে।

## কেন মূল্যায়ন-নিয়ন্ত্রিত উন্নয়ন?

AI এজেন্ট তৈরি করার সময়, "এটা ঠিকঠাক দেখাচ্ছে" যথেষ্ট নয়। **মূল্যায়ন-নিয়ন্ত্রিত উন্নয়ন** এজেন্ট আউটপুটকে কোডের মতো বিবেচনা করে: প্রথমে পরীক্ষা লেখেন, গুণমান মাপেন, এবং শুধুমাত্র স্কোর একটি নির্দিষ্ট মান পূরণ করলে শিপ করেন।

Zava Creative Writer (অংশ ৭)-এ, **সম্পাদনাকারী এজেন্ট** ইতোমধ্যে একটি হালকা ওজনের মূল্যায়নকারী হিসেবে কাজ করে; এটি ACCEPT বা REVISE সিদ্ধান্ত নেয়। এই ল্যাবটি ঐ প্যাটার্নকে একটি পুনরাবৃত্তিযোগ্য মূল্যায়ন কাঠামোতে রূপান্তর করে যা আপনি যেকোন এজেন্ট বা পাইপলাইনে প্রয়োগ করতে পারেন।

| সমস্যা | সমাধান |
|---------|----------|
| প্রম্পট পরিবর্তন গুণমান নীরবভাবে ভঙ্গ করে | **সোনালী ডেটাসেট** পেছনে যাওয়া ধরতে পারে |
| "একটি উদাহরণে কাজ করে" পক্ষপাতিত্ব | **বহু টেস্ট কেস** আভিজাত্য প্রকাশ করে |
| বিষয়ভিত্তিক গুণমান মূল্যায়ন | **নিয়ম-ভিত্তিক + LLM-যেমন-নির্ণায়ক স্কোরিং** সংখ্যা প্রদান করে |
| প্রম্পট বৈচিত্র্য তুলনা করার কোন উপায় নেই | **পার্শ্ব-পর্শ্ব মূল্যায়ন রান** সমন্বিত স্কোর সহ |

## মূল ধারণাসমূহ

### ১. সোনালী ডেটাসেট

একটি **সোনালী ডেটাসেট** হলো পরিচর্যা করা টেস্ট কেসের সেট যার প্রত্যাশিত আউটপুট জানা থাকে। প্রতিটি টেস্ট কেস অন্তর্ভুক্ত করে:

- **ইনপুট**: এজেন্টকে পাঠানোর জন্য প্রম্পট বা প্রশ্ন
- **প্রত্যাশিত আউটপুট**: সঠিক বা উচ্চমানের উত্তর যা কীওয়ার্ড, গঠন, তথ্য ইত্যাদি থাকতে পারে
- **বিভাগ**: প্রতিবেদন তৈরির জন্য গ্রুপিং (যেমন "তথ্যগত নির্ভুলতা", "স্বর", "সম্পূর্ণতা")

### ২. নিয়ম-ভিত্তিক চেকস

দ্রুত, নির্ধারিত পদ্ধতিতে পরীক্ষা যা LLM এর প্রয়োজন হয় না:

| চেক | এটি কী পরীক্ষা করে |
|-------|--------------|
| **দৈর্ঘ্য সীমা** | উত্তর খুব ছোট (কর্মহীন) বা খুব বড় (অবিচ্ছেদপূর্ণ) নয় |
| **প্রয়োজনীয় কীওয়ার্ড** | উত্তর প্রত্যাশিত শব্দ বা সত্তাগুলো উল্লেখ করে |
| **ফর্ম্যাট যাচাই** | JSON পার্সযোগ্য, Markdown হেডার উপস্থিত |
| **নিষিদ্ধ বিষয়বস্তু** | কোন বিভ্রান্তিকর ব্র্যান্ড নাম নেই, কোন প্রতিযোগীর উল্লেখ নেই |

### ৩. LLM-যেমন-নির্ণায়ক

একই লোকাল মডেল ব্যবহার করে নিজের আউটপুট (অথবা ভিন্ন প্রম্পটের আউটপুট) গ্রেড করা। বিচারক পায়:

- মূল প্রশ্ন
- এজেন্টের উত্তর
- গ্রেডিং মানদণ্ড

এবং একটি সংগঠিত স্কোর ফেরত দেয়। এটি অংশ ৭ থেকে সম্পাদক প্যাটার্নের মতোই, কিন্তু একটি টেস্ট স্যুটের ওপর সিস্টেম্যাটিকভাবে প্রয়োগ করা হয়।

### অনুশীলন ২ - সোনালী ডেটাসেট বোঝা

মূল্যায়ন স্যাম্পলে সংজ্ঞায়িত টেস্ট কেসগুলো পরীক্ষা করুন। প্রতিটি টেস্ট কেসে রয়েছে:

**প্রশ্নগুলো ভাবুন:**
1. কেন প্রত্যাশিত মানগুলো **কীওয়ার্ড** সম্পূর্ণ বাক্যের পরিবর্তে?
2. নির্ভরযোগ্য মূল্যায়নের জন্য কতগুলো টেস্ট কেস প্রয়োজন?
3. আপনি আপনার নিজস্ব অ্যাপ্লিকেশনে কোন কোন বিভাগ যোগ করবেন?

### অনুশীলন ৩ - নিয়ম-ভিত্তিক বনাম LLM-যেমন-নির্ণায়ক স্কোরিং বোঝা

মূল্যায়ন কাঠামো ব্যবহার করে **দুটি স্কোরিং স্তর**:

#### নিয়ম-ভিত্তিক চেকস (দ্রুত, নির্ধারিত)

```
✓ Length: 50-500 words       → 1 point
✓ Keywords: 3/4 found        → 0.75 points  
✗ Forbidden: mentions "Home Depot" → 0 points
─────────────────────────────
Rule score: 0.58 / 1.0
```

#### LLM-যেমন-নির্ণায়ক (সুশৃঙ্খল, গুণগত)

সেই একই লোকাল মডেল একটি নির্দিষ্ট রুব্রিক সহ বিচারক হিসাবে কাজ করে:

```
Rate this response on a scale of 1-5:
- 1: Completely wrong or irrelevant
- 2: Partially correct but missing key information
- 3: Adequate but could be improved
- 4: Good response with minor issues
- 5: Excellent, comprehensive, well-structured

Score: 4
Reasoning: The response correctly identifies all necessary tools
and provides practical advice, but could include safety equipment.
```

**প্রশ্নগুলো ভাবুন:**
1. কখন আপনি নিয়ম-ভিত্তিক যাচাইকরণকে LLM-যেমন-নির্ণায়কের চেয়ে বেশি বিশ্বাস করবেন?
2. একটি মডেল কি নিজের আউটপুট নির্ভরযোগ্যভাবে বিচার করতে পারে? সীমাবদ্ধতাগুলো কী কী?
3. এটি অংশ ৭ এর সম্পাদক এজেন্ট প্যাটার্নের সাথে কীভাবে মিল?

### অনুশীলন ৫ - আপনার নিজের এজেন্টে মূল্যায়ন প্রয়োগ করা

মূল্যায়ন কাঠামোকে আপনার নিজের এজেন্টের জন্য টেম্পলেট হিসেবে ব্যবহার করুন:

1. **আপনার সোনালী ডেটাসেট সংজ্ঞায়িত করুন**: প্রত্যাশিত কীওয়ার্ড সহ ৫-১০ টি টেস্ট কেস লিখুন।
2. **আপনার সিস্টেম প্রম্পট লিখুন**: এজেন্ট নির্দেশনাগুলো যা আপনি পরীক্ষা করতে চান।
3. **মূল্যায়ন চালান**: বেসলাইন স্কোর পান।
4. **পুনরাবৃত্তি করুন**: প্রম্পট সামঞ্জস্য করুন, পুনরায় চালান, এবং তুলনা করুন।
5. **একটি থ্রেশহোল্ড নির্ধারণ করুন**: যেমন "০.৭৫ এর কম সম্মিলিত স্কোরে শিপ করবেন না"।

### অনুশীলন ৬ - Zava সম্পাদক প্যাটার্নের সাথে সংযোগ

Zava Creative Writer-এর সম্পাদক এজেন্টকে পুনরায় দেখুন (`zava-creative-writer-local/src/api/agents/editor/editor.py`):

```python
# সম্পাদকটি প্রোডাকশনে একটি এলএলএম-অ্যাস-জাজ।
{"decision": "accept/revise", "editorFeedback": "...", "researchFeedback": "..."}
```

এটি অংশ ৮ এর LLM-যেমন-নির্ণায়ক এর **একই ধারণা**, তবে অফিসলাইনের টেস্ট স্যুট নয়, প্রোডাকশন পাইপলাইনে এম্বেড করা। উভয় প্যাটার্ন:

- মডেল থেকে সংযুক্ত JSON আউটপুট ব্যবহার করে।
- সিস্টেম প্রম্পটে সংজ্ঞায়িত গুণমান মানদণ্ড প্রয়োগ করে।
- পাস/ফেল সিদ্ধান্ত নেয়।

**পার্থক্য:** সম্পাদক প্রোডাকশনে (প্রতিটি অনুরোধে) চলে। মূল্যায়ন কাঠামো ডেভেলপমেন্টে (শিপ করার আগে) চলে।

## পরবর্তী ধাপ

- **স্কেল আপ করুন**: আপনার সোনালী ডেটাসেটে আরও টেস্ট কেস ও বিভাগ যোগ করুন।

- **সময় অনুসারে ট্র্যাক করুন**: মূল্যায়ন ফলাফল সংরক্ষণ করুন যেন প্রম্পট ও মডেল সংস্করণের তুলনা করা যায়।
