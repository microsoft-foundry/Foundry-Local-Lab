# অংশ ১৩: কর্মশালা সম্পূর্ণ

> **অভিনন্দন!** আপনি সম্পূর্ণ Foundry Local কর্মশালা সফলভাবে সম্পন্ন করেছেন।

আপনি CLI ইনস্টল করা থেকে শুরু করে চ্যাট অ্যাপ্লিকেশন তৈরি করা, RAG পাইপলাইন, মাল্টি-এজেন্ট সিস্টেম, স্পিচ-টু-টেক্সট ট্রান্সক্রিপশন, নিজস্ব কাস্টম মডেল কম্পাইল করা, টুল কলিং সক্ষম করা, এবং মাল্টি-এজেন্ট অ্যাপ্লিকেশনের জন্য লাইভ ওয়েব UI সংযোগ করা পর্যন্ত করেছেন। সবকিছুই সম্পূর্ণভাবে আপনার ডিভাইসে চলছে।

---

## আপনি যা তৈরি করেছেন

| ২ | Foundry Local SDK API দক্ষতা অর্জন করেছেন সার্ভিস, ক্যাটালগ, ক্যাশ এবং মডেল ব্যবস্থাপনার জন্য |
| ৩ | SDK ব্যবহার করে Python, JavaScript, এবং C# থেকে OpenAI সঙ্গে সংযোগ স্থাপন করেছেন |
| ৪ | স্থানীয় জ্ঞান পুনরুদ্ধারের জন্য RAG পাইপলাইন তৈরি করেছেন |
| ৫ | পার্সোনা ও স্ট্রাকচার্ড আউটপুট সহ AI এজেন্ট তৈরি করেছেন |
| ৬ | প্রতিক্রিয়া লুপ সহ মাল্টি-এজেন্ট পাইপলাইন পরিচালনা করেছেন |
| ৭ | প্রোডাকশন ক্যাপস্টোন অ্যাপ্লিকেশন, Zava Creative Writer, অনুসন্ধান করেছেন |
| ৮ | এজেন্টদের জন্য মূল্যায়ন-নির্দেশিত উন্নয়ন কর্মপ্রবাহ তৈরি করেছেন |
| ৯ | Whisper ব্যবহার করে অডিও ট্রান্সক্রাইব করেছেন, ডিভাইসে স্পিচ-টু-টেক্সট সম্পন্ন করেছেন |
| ১০ | কাস্টম Hugging Face মডেল ONNX Runtime GenAI দিয়ে কম্পাইল এবং রান করেছেন |
| ১১ | স্থানীয় মডেলকে টুল কলিং দিয়ে বহির্গত ফাংশন কল সক্ষম করেছেন |
| ১২ | Zava Creative Writer-এর জন্য রিয়েল-টাইম স্ট্রিমিং সহ ব্রাউজার-ভিত্তিক UI তৈরি করেছেন |

## আরও ধারণা

- টুল কলিং এবং এজেন্ট একত্র করলে স্বয়ংক্রিয় কর্মপ্রবাহ তৈরি করুন
- আপনার টুল থেকে স্থানীয় ডাটাবেজ বা অভ্যন্তরীণ REST API কল করুন
- বিভিন্ন মডেল (`phi-4-mini`, `deepseek-r1-7b`) চেষ্টা করে গুণমান এবং গতি তুলনা করুন
- আপনার Zava Writer ওয়েব সার্ভারে প্রমাণীকরণ বা রেট সীমাবদ্ধতা যোগ করুন
- আপনি যত্নশীল এমন একটি ডোমেইনের জন্য নিজস্ব মাল্টি-এজেন্ট অ্যাপ্লিকেশন তৈরি করুন
- Foundry Local এর পরিবর্তে Azure AI Foundry ব্যবহার করে ক্লাউডে ডিপ্লয় করুন, একই কোড, ভিন্ন এন্ডপয়েন্ট
