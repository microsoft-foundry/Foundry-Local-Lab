# ส่วนที่ 13: เวิร์กช็อปครบสมบูรณ์

> **ขอแสดงความยินดี!** คุณได้ทำเวิร์กช็อป Foundry Local ฉบับเต็มเสร็จเรียบร้อยแล้ว

คุณได้ผ่านตั้งแต่การติดตั้ง CLI ไปจนถึงการสร้างแอปแชท, ท่อ RAG, ระบบตัวแทนหลายตัว, การถอดความเสียงเป็นข้อความ, การคอมไพล์โมเดลที่กำหนดเอง, การเปิดใช้งานการเรียกเครื่องมือ, และการเชื่อมต่อ UI เว็บสดสำหรับแอปตัวแทนหลายตัวของคุณ ทั้งหมดนี้ทำงานได้บนอุปกรณ์ของคุณเอง

---

## สิ่งที่คุณสร้างขึ้น

| ส่วน | สิ่งที่คุณสร้างขึ้น |
|------|---------------------|
| 1 | ติดตั้ง Foundry Local, สำรวจโมเดลผ่าน CLI |
| 2 | เรียนรู้ API ของ Foundry Local SDK สำหรับบริการ, แคตตาล็อก, แคช และการจัดการโมเดล |
| 3 | เชื่อมต่อจาก Python, JavaScript, และ C# โดยใช้ SDK กับ OpenAI |
| 4 | สร้างท่อ RAG ด้วยการดึงความรู้ในเครื่อง |
| 5 | สร้างเอเย่นต์ AI พร้อมบุคลิกและผลลัพธ์ที่มีโครงสร้าง |
| 6 | ประสานงานท่อเอเย่นต์หลายตัวพร้อมวงจรป้อนกลับ |
| 7 | สำรวจแอปพลิเคชันหม่งผลิตในงานต้นแบบ, Zava Creative Writer |
| 8 | สร้างกระบวนการพัฒนาที่นำด้วยการประเมินสำหรับเอเย่นต์ |
| 9 | ถอดความเสียงด้วย Whisper, การพูดเป็นข้อความบนอุปกรณ์ |
| 10 | คอมไพล์และรันโมเดล Hugging Face ที่กำหนดเองด้วย ONNX Runtime GenAI |
| 11 | เปิดใช้งานโมเดลในเครื่องให้เรียกฟังก์ชันภายนอกด้วยการเรียกเครื่องมือ |
| 12 | สร้าง UI บนเบราว์เซอร์สำหรับ Zava Creative Writer พร้อมการสตรีมแบบเรียลไทม์ |

---

## ไอเดียเพิ่มเติม

- ผสมผสานการเรียกเครื่องมือกับเอเย่นต์เพื่อสร้างเวิร์กโฟลว์อัตโนมัติ
- สืบค้นฐานข้อมูลในเครื่องหรือติดต่อ REST API ภายในจากเครื่องมือของคุณ
- ทดลองใช้โมเดลต่าง ๆ (`phi-4-mini`, `deepseek-r1-7b`) เพื่อเปรียบเทียบคุณภาพและความเร็ว
- เพิ่มการตรวจสอบสิทธิ์หรือจำกัดอัตราการใช้งานสำหรับเว็บเซิร์ฟเวอร์ Zava Writer ของคุณ
- สร้างแอปตัวแทนหลายตัวที่คุณสนใจในโดเมนของคุณเอง
- ปรับใช้บนคลาวด์โดยเปลี่ยน Foundry Local เป็น Azure AI Foundry ใช้โค้ดเดียวกันแต่จุดเชื่อมต่อแตกต่างกัน
